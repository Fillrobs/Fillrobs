Managing Network Virtualization

Once a Network Virtualization technology is configured, begin associating one or more vCenter Resource Handlers to the NSX-T Manager. Currently, CloudBolt only supports vCenter.

Setup Steps

  1. Ensure at least one vCenter Resource Handler exists in your CloudBolt instance.
  2. Refresh Mappings to notify CloudBolt about NSX-T's compute managers.
  3. Import Logical Routers to handle traffic between internal and external networks.
  4. Import Transport Zones to determine which VMs can access and use a particular network.

Overview Tab

View the currently configured Network Virtualization technologies, or take the following actions:

  • Click Edit to change the connection information.
  • Click Delete to remove the Network Virtualization connection.
  • Click Verify Connection to make an API call to the technology and validate the connection. 


Resource Handlers

CloudBolt's integration with NSX-T currently only supports vCenter Resource Handlers. You must have at least one vCenter Resource Handler configured to use this feature.
The Resource Handler you associate with NSX-T cannot include Environments using NSX-V. 

Refresh Mappings

NSX-T maintains a list of compute managers, such as vCenter, to which it is connected and for which it can provide network resources. Refreshing Mappings notifies CloudBolt about the compute managers this NSX-T instance is aware of.

On the Resource Handlers tab of your NSX-T configuration, click Refresh Mappings > Refresh. The Resource Handlers tab will list available vCenter options.


Associating Resource Handler(s)

Once the Mapping Reference(s) is visible, click the pencil icon in the Resource Handler column to associate a vCenter Resource Handler with this NSX-T configuration.


Logical Routers

Logical Routers handle traffic between internal and external networks, such as between a datacenter and the internet. In NSX-T, routing topologies are tiered to allow multiple tenants control over their own routing services and policies.

There are two levels of Logical Routers: Tier-0 and Tier-1.
  • Tier-0: This top-tier router connects to the physical network at the north end of Tier-0 interfaces, and southbound to Tier-1.
  • Tier-1: This tier connects northbound to Tier-0 for public network access, and southbound to Logical Switches.

Importing Logical Routers

To import Logical Routers, at least one must exist in your NSX-T Data Center.

  1. On the selected NSX-T connection, click the Logical Routers tab.
  2. Click Import Logical Routers. The available Logical Routers will appear in the Logical Routers tab with a success message.
  1. In the Actions column, click the pencil icon to associate an Environment to a Logical Router. Click X to delete a Logical Router.

Transport Zones

Transport zones determine which hosts and, consequently, which VMs can access and use a particular network. This is done by the transport zone limiting which hosts can "see" a logical switch, therefore limiting the VMs that can attach to the logical switch.

A host can belong to more than one transport zone; a logical switch can only belong to one transport zone.

Importing Transport Zones

In order to import transport zones, at least one must exist in your NSX-T Data Center.

  1. On the selected NSX-T connection, click the Transport Zones tab.
  2. Click Import. The available transport zones will appear in the Transport Zones tab with a success message.
  1. In the Actions column, click the pencil icon to associate an Environment to a Transport Zone. Click X to delete the Transport Zone.
