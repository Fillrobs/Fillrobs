CloudBolt Appliance Security

Follow your organization's Information Security best practices when protecting the CloudBolt platform.

Introduction

CloudBolt is a highly sensitive piece of infrastructure, as it will often house credentials with elevated rights into multiple cloud environments. The CloudBolt appliance can be configured to meet differing hardening guidelines, depending on your regulatory environment. If you have specific hardening needs, contact CloudBolt Customer Success.

TLS and End to End Transport Encryption

Avoid unencrypted protocols during deployment, when establishing connections to external providers (e.g. Resource Handlers, Configuration Managers, and Provisioning Engines).

Ensure that SSL verification is enabled globally and on each connection to ensure you are protected against Man in the Middle attacks on TLS encrypted connections.

Network Segmentation and Firewalls

Consider using network controls to limit exposure of the CloudBolt services to the wider corporate network as part of following best practice security guidelines.

Network controls can be used to limit connectivity to and from the appliance to ensure only essential traffic is permitted. See Network Connectivity for details. 

Audit Logging and SIEM

Leverage CloudBolt's SIEM integration and Multi-Channel Alerts to send events outside of the CloudBolt platform to ensure integrity of audit logs.

Single Sign-On and Multi-Factor Authentication

We recommend deploying CloudBolt's SAML-based SSO rather than LDAP based authentication where possible. Using single sign on reduces password fatigue and susceptibility to phishing. 

When deploying SSO, leverage your Identity Provider's (IdP) conditional access policy where available, to ensure that multi-factor authentication is enforced when necessary.

If you are using Azure AD (Office365) or Google Workspace (formerly G Suite), you may be able to leverage these directly for SSO. They provide the necessary SAML services if you are not using an auxiliary identity provider solution.
