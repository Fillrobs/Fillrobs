Guided Setup

Use Guided Setup to quickly configure your CloudBolt instance.

After CloudBolt is downloaded and installed, you will be taken through the Guided Setup feature. Once an Admin Account is created, you can skip or exit the wizard if desired.

Initial Login

Use the default credentials shown to launch Guided Setup.

Default Username        Default Password       
admin
  • Default password for OVA builds is: admin
  • Default password for deploying from AWS Marketplace is the AWS Instance ID

Accept EULA

Read the End User License Agreement fully, and only proceed if you accept the terms.

Proxy Setup 

CloudBolt supports using a proxy to connect to external systems such as public cloud services, GitHub, ServiceNow, CloudBolt Updates, and more.

  • Fill out the required fields and click Set Proxy.
  • If you choose not to set up a proxy at this time, choose No Proxy.
SSL certificates are required to access the Content Library when using a proxy in your network.

Create CloudBolt Admin Account

The account created in this step will replace the default credentials from Initial Login.

Field name Field description
First Name          Enter the first name for the person who will own this account.       
Last Name Enter the last name for the person who will own this account.       
Username Create a username for this Admin account.
Email Enter the email address that will link to this Admin account.  
Password/Confirm Password        Create a strong* password. The entry here cannot contain the word "Password".
Password Reset Question Click the dropdown to choose a Password Reset Question.
Password Reset Answer Enter an answer for the question chosen above.
Group Name for Users Enter a Group name for Users.

*password requirements:

  • 9 character minimum.
  • Cannot contain the word "Password".
  • Three out of four categories: uppercase, lowercase, special character, symbol.

Select a Provider or Jump Right into CloudBolt

  • If vCenter, AWS, or Azure is chosen, you will continue to Cloud Provider Setup below.
  • Choosing another provider will take you to the Create a Resource Handler page for that selection.

View specific setup steps for your provider:

vCenter Provider

  1. Authorize vCenter. Supply administrative credentials for the vCenter account. CloudBolt will use the authorized vCenter account to perform server creation, power management, synchronization, deletion, and other management actions.
  2. Import Clusters. This list is generated from your vCenter Server. They are required in order to deploy using CloudBolt.
  3. Import Templates. Select the vCenter templates you wish to import into CloudBolt. This list is generated from your vCenter Server. At least one template must be selected in order to provision VMs.
  4. CloudBolt requires at least one network set as the default for provisioning servers in order to set the IP and hostname. Select one from the list and click Connect Default Network.
  5. Customize Environments for vCenter, or click Save to use the defaults. Defaults can always be changed on the Environment later.
  6. Start using CloudBolt.
    • ​Build Servers
    • Add Groups
    • Visit the Dashboard
    • Configure Admin settings
    • Interact with discovered resources, which are the Servers on the connected vCenter Server. 
    • Add Users

AWS Provider

  1. Authorize AWS. Enter your Access Key ID and Secret Access Key to set up Environments for AWS and easily deploy and customize AWS instances from within CloudBolt. See AWS for information about these credentials. 
  2. Import Regions. Select up to five initial Regions to import as CloudBolt Environments. AWS Regions are required to deploy using CloudBolt. 
  3. Import AMIs. Select at least one Image to import into CloudBolt. 
  4. Customize Environments for AWS, or click Save to use the defaults. Defaults can be changed on the Environment later.
  5. Start using CloudBolt.
    • ​Build Servers
    • Add Groups
    • Visit the Dashboard
    • Configure Admin settings
    • Interact with discovered resources, which are the Servers on the connected vCenter Server. 
    • Add Users

Azure Provider

  1. Authorize Azure. Enter the IDs and Key that CloudBolt uses to connect to the Azure Resource Manager. See Azure for information on these credentials. 
  2. Import Locations. Select up to five initial Locations to import as CloudBolt Environments. Azure Locations are required to deploy using CloudBolt.
  3. Import Images. Select at least one Image to import into CloudBolt.
  4. Customize Environments for Azure, or click Save to use the defaults. Defaults can always be changed on the Environment later.

  5. Start using CloudBolt.
    • ​Build Servers
    • Add Groups
    • Visit the Dashboard
    • Configure Admin settings
    • Interact with discovered resources, which are the Servers on the connected vCenter Server. 
    • Add Users
