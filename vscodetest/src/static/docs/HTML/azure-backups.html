Azure Backup Data Protection in CloudBolt

Learn how to use an Azure Backup Data Protection Plan in your CloudBolt instance.

Introduction

CloudBolt uses the existing Data Protection Plan feature to introduce a new integration: Azure Backup. Attach the Data Protection Parameter to your Environment, Blueprint, or Server and select Azure Backup to begin protecting your Azure Servers from data loss.

This first iteration of Azure Backup integration should be considered a working beta. Help improve this integration by reporting any issues to support@cloudbolt.io.

Prerequisites

  • An existing Azure Resource Handler with at least one Region / Image imported.
  • An Azure Backup Vault and Policy configured in your Azure Portal.
  • Required Azure Roles: See Which permissions are required to enable backup for a VM?

Configure Backup in Azure

Create a Backup Vault

  1. In your Azure Portal, search for Backup center and click the Backup center result to open it.
  2. Under Manage, click Vaults > + Vault.
  3. Select Recovery Services vault > Continue.
  4. Select your Subscription and a Resource Group, then create a Vault name and select a Region.
  5. Click Create.

Create a Backup Policy

  1. In the Backup center, click Backup policies > + Add.
  2. Select Azure virtual machines as the Datasource type and select the Vault you created above.
  3. Click Continue.
  4. Enter the Policy Name and set up the desired Backup schedule.
    1. Retention range: The retention of daily backup point refers to how long the snapshots are retained before you are allowed to delete them. Set this number to however long you would like snapshots to last before they can be deleted.
  5. Click Create. 
  6. Continue to Configure Azure Backup Data Protection in CloudBolt. 

Configure Azure Backup Data Protection in CloudBolt

  1. Follow the steps outlined in Adding a Data Protection Plan and Data Protection Plan Parameters, selecting Azure as your technology.
  2. On an Azure Server with the Data Protection Parameter, view the Data Protection tab.
    1. The Protection Plan number correlates to the Backup Job ID.
  3. Click the Actions button to send a request to Azure to restore a host from backup. Note that this does not currently create a Job in CloudBolt's Job Engine. 
Do not click the Action button more than once. After a few moments, a banner message will confirm that CloudBolt successfully sent a backup request to Azure.

Outcomes

Your Azure Backup is ready to protect your Servers from data loss.
