Deploy Multiple Images from One Blueprint

Learn how to set up a single CloudBolt Blueprint to deploy all of your Cloud Platform images.

Introduction

Blueprints are a very powerful feature in the CloudBolt platform. You can orchestrate the provision, decommission, and sync states of your applications. They can be used to deploy images on multiple cloud platforms. To reduce splay, you can configure a single Blueprint to deploy all of your Cloud Platform images.

Prerequisites

  • To succeed, a general understanding of CloudBolt OS Builds and Blueprints is required.
  • Create the Blueprint you wish to use.

Deploying Multiple Images on One Blueprint

  1. In CloudBolt, create an OS Build and add the desired Images to it.
  2. Navigate to the Blueprint.
  3. On the Build tab, Add a Server Tier to the Blueprint. Choose the OS Build you configured in Step 1. 
  4. On the Edit Blueprint Item form, leave the Allowable OS Families field empty.

  5. Click Save.
  6. Click the caret to expand the Server Tier Build Item. Under Enabled Environments, toggle the button to All Capable.

Outcomes

When this Blueprint is used in an Order, CloudBolt will display all of the valid Environments for the selected OS Build to deploy a Server from.
