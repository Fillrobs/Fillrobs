Restarting the Job Engine

Manually restarting the Job Engine

When manually restarting the Job Engine, currently running jobs will be killed and stuck in RUNNING state.

  1. To manually restart the job engine, use Supervisord to kill and restart the job engine workers:
  • Copy Code
    Python
    supervisorctl restart jobengine:*
  1. Supervisord should then restart a jobengine.py process and new pending jobs should start running as expected.

Enable the Job Engine to Forcibly Restart


Introduction

Jobs that run for longer than the max timeout are gently canceled, given a "Canceling" status, and have an exception inserted into their thread to cancel the Job.

This method might not work when a thread's control is not in a Python block and therefore might never be canceled. When this happens, the Job Engine could not restart, which would often mean it grows in memory over the course of weeks until the main thread gets hung and jobs no longer execute.


Solution

When Job Engine Jobs are stuck in a non-completed state for more than 2x the max global job timeout (set in Admin > Miscellaneous Settings) and there are no other active jobs running, the Job Engine will kill those threads and forcibly restart itself, removing hung jobs in the process.

The soft cancellation (no forced restart) Job Engine behavior will remain in place and act as the default behavior for individual jobs that are able to be canceled before 2x the max global job timeout.

If the restart is successful, the Job Engine will run again as normal.

Enable the Job Engine to Forcibly Restart

This feature will be enabled by default in all versions after version 9.4.7.

To enable this feature in 9.4.7, run the following commands on your desired instance. 

Once enabled, no further setup is required in CloudBolt. 

Copy Code
Code
ssh root@your.ip
cd /opt/cloudbolt
python manage.py features -e "jobengine::hard_reset_past_timeout"


Email Alert

If the forced Job Engine restart is not successful and Jobs continue to run past 3x the max global timeout, an email will be sent to the CB Admins with a description of the Job that caused the issue.

  • Configure the email's "From" address in Admin > System > Email Settings.
  • The email's subject and body will automatically populate. The subject will be NOTICE: Long-running Job Engine.
  • The body will contain the Job name(s) that is still running 3x past the max global timeout.
