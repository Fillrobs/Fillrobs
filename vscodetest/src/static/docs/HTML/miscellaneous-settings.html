Miscellaneous Settings

Miscellaneous Settings is a place to set Global settings and customize areas of CloudBolt like error messages, whitelists, security messages, and more.


Customize Miscellaneous Settings

To access Miscellaneous Settings, click Admin > System > Miscellaneous Settings.

Setting Name Description
New Server button Enabled by default. If disabled, the "New Server" button will not be visible except to CloudBolt Admin users. This limits server orders through the Catalog only, for non-admin users. 
Password Toggle Disabled by default. If enabled, users can unmask passwords in any form field if they have the appropriate permissions to do so.
Original Password Verification Enabled by default. Changing a user's password requires entering their previous password.
Password Reset Question When enabled, new users will be asked to set a challenge question and answer. Disable when using features such as Active Directory to allow already authenticated users to bypass the challenge question during login.
Require New Users to Self-Assign Passwords
Enable to allow Users to designate their own passwords when an account is created for them, rather than the Admin designating the password. See Creating a New User.
Require User Email Verification Enabled by default. Users must verify their email address via an emailed confirmation link after the user's email address has changed.
Social Disabled by default. When enabled, CloudBolt users can post to social websites.
Avatars

Enabled by default. Avatar images provided by gravatar.com. Users must create a Gravatar profile and associate an image with their email address. 

Restrict new environments Enabled by default. When enabled, new CloudBolt Environments are restricted to only the Unassigned Group (and therefore, they will not show on the order form until they are available to more Groups).
Reuse historical hostnames
When enabled, hostnames of servers that are no longer active will be reused when generating unique hostnames during server provisioning.
Fast Track Server Deletion
When enabled, start decommissioning jobs immediately instead of routing requests through the normal order approval process.
Enforce single sessions for users
If a user logs in from a second location or browser, the first session is logged out.
Show rates when ordering
Display rate information when ordering servers.
Show video tips
Display video tips in the user interface when the video icon is clicked.
Show cost preview when ordering
Display a cost preview per Environment in the Order Form.
Allow exceeding quotas
Enable to allow exceeding quotas.
Hostnames case sensitive
Enable to allow the use of the same hostname with different cases (server01 and SERVER01).
Inherit Group Parameters
Enable to allow parameters and preconfigurations set on parent groups to inherit down to all of their subgroups. This includes any options and constraints set on the parent group.
Automatically login to SSO
If the is an SSO Provider configured for the portal, automatically redirect users there for login.
Embedded Mode

When enabled, non-admin users can only access CloudBolt pages with ?frame=true appended to the URL. Use this feature when nesting CloudBolt UI in an iFrame, such as ServiceNow Integration.

Select the Global Role(s) in Allowed Global Roles in Embedded Mode to allow access on a more granular level: Super Admin, API Access, DevOps Admin, or Global Viewer.
Valid group types for Orders
Enter all group types you wish to be valid for Orders here. Click the star to select all items.
Security message
Text entered here is shown under the navigation bar on all pages. Up to 1000 characters allowed. Valid HTML tags include (<a>, <b>, <i>, <span>, <br>, <h1>, <h2>, <h3>, <h4>, <h5>, <h6>). 
Legal Notice Text
Admin access only. Enter text here that will display to Users upon login and Save Changes. The text will appear the next time a User logs in. This text requires the "Dismiss" action to proceed into CloudBolt. Leave this field blank to disable this feature.
Display Legal Notice on Every LoginEnabled by default. This setting will show the Legal Notice Text upon every login. Disable to only show Legal Notice Text once, the first time the User logs in after text is saved in the above field.
URL for site-specific help
The URL entered here overrides the 'help' link in CloudBolt's navigation bar.
CB Admin E-mail
Enter an email address here for the CloudBolt admin.
Generic error message
A message that is shown on failed job pages, HTTP error pages (500, 404, etc), and other pages where the user hit a problem. This can be used to provide the user with contact info on how to get support for CloudBolt within your company. 
Default redirect URL to redirect users users to when they navigate to / on the CB server. Common examples are '/dashboard/' (the default), '/catalog/', and '/servers/'. 
Inactivity timeout minutes
If set, log user out after this many minutes of inactivity. Note that a value of 0 means no timeout. 
Remote script *nix directory
The directory that will be used when writing remote scripts to *nix VMs.
Remote script Windows directory
The directory that will be used when writing remote scripts to Windows VMs via VMWare Tools.
Default Addressing Scheme
Controls whether to use the IPv4 or Ipv6 address for a Server, when both are known. 
Bypass proxy domains
A comma separated list of the domains for which CloudBolt should not attempt to use a proxy server to route the connections. These domain names are regular expressions and could also be represented as ipv4 IP addresses. Ex. '.*\.internal\.loc,192\.168\..*,my\.domain\.com'. 
External URL Whitelist
A semicolon-separated list of the domains to which WebHooks and CloudBolt Actions using source code from an external URL are allowed to connect. These domain names are regular expressions and could also be represented as ipv4 IP addresses. Ex. '.*\.internal\.loc;192\.168\..*;my\.domain\.com'. Defaults to allowing everything: '.*' 
Embed URLs / Frame Ancestors A space-separated list of URLs which are allowed to frame the CloudBolt UI, such as ServiceNow iFrame Integration. This list can use a mix of formats: "example.com" / "*.example.com" / "https://*.example.com:12/path/to/file.js" 
Default job timeout
The maximum number of seconds that all jobs are limited to. The default is 86400 seconds.
Default User Display Scheme
This controls how user information is displayed when changing owner on a resource. 
Group name levels to show

How many levels of ancestors' names to show when this name is shown outside the context of the group hierarchy.

Default Catalog Viewing Mode
This controls how Blueprints will be organized and styled when browsing the Catalog page.
Show "Recipient" field on order form
Choose from When Allowed, Never, or Only Enabled Blueprints. Controls when the "Recipient" field is shown to permitted users on the order form. If set to Only Enabled Blueprints, Blueprint managers can access the per-Blueprint setting using the Edit button on the Blueprint's Overview tab. If the field is not shown, the user ordering the Blueprint will be the sole recipient.
Console Feature
Choose resource handlers to enable the console feature on.
Terraform Version
The version of Terraform used to execute Terraform Plan Blueprint actions.
RDP/SSH Terminal
Choose Resource Handlers on which the RDP/SSH terminal feature should be enabled.
Disabled out-of-the-box reports
Choose which out-of-the-box reports to disable for all users.
Disabled Recent Activity categories
Choose which recent activity history categories to disable for all users.
Web Proxy
If desired, choose a proxy server to use for outgoing connections (to public cloud providers like Azure, etc.).
