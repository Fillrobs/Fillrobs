Configure OneLogin SSO

Learn how to configure OneLogin SSO in CloudBolt, as well as account and application configuration in OneLogin.


Considerations

  • An Administrator account must exist in OneLogin that can create applications and add Users.
  • A OneLogin organization must exist or be created.

Set up OneLogin in CloudBolt

  1. Log in to CloudBolt as an Admin user.
  2. Click Admin > Security > See All.
  3. Click Single Sign-On (SSO).
  4. Click Add a Single Sign-On IdP > Generic SAML Provider.
  5. Complete the Configure a SAML Provider form.
Note: You will need the Single Sign-On URL and Metadata URL / Entity ID to use in the OneLogin Application setup below.

Create the OneLogin Application

In OneLogin, make sure you are in the Admin menu (https://<instance-name>.onelogin.com/admin).

  1. In the main navigation menu, click the Applications Tab.
  2. Click Add App.
  3. In the Search box, type "SAML Test" and Search.
  4. Select "SAML Test Connector (Advanced)".
  5. Enter a Display Name for the Portal and ensure "Visible in portal" is checked.

Configuration Settings

  1. Configure the SAML settings to follow the table below.
  • Metadata URL / Entity ID: Find this field in CloudBolt, on the OneLogin Single Sign-On provider Overview page.
  • Single Sign-On URL: Find this field in CloudBolt, on the OneLogin Single Sign-On provider Overview page.

See Configuring the Identity Provider.

The name ID for the email, given name, surname, and user ID fields are specified as parameters within the application (see below).


Download the SAML Metadata File

  1. On the newly created application, click the More Actions dropdown in the top right.
  2. Select "SAML Metadata" to automatically trigger file download.


Assign User Access to the Applications

  1. In the main navigation menu, click Users tab, and select the desired User from the list.
  2. In the left navigation pane, click Applications.
  3. Click [+] in the top right.
  4. Select the Application and the Role to assign access to the User.
  5. Click Save User.

Complete CloudBolt SSO Authentication for OneLogin

  1. Log in to CloudBolt using a Super Admin role account.
  2. Click Admin > Security > See All.
  3. Select Single Sign-On (SSO).
  4. Select OneLogin.
  5. Click Import XML Metadata and choose the XML File downloaded in Download the SAML Metadata File.
  6. Click the Edit icon.
  7. Scroll to the Branded Portal dropdown and select a Portal to use for Okta if one was not chosen at setup time.
  8. (Optional) Enable "Automatically Login to SSO".
    1. Click Admin > Miscellaneous Settings.
    2. Scroll to "Automatically login to SSO" and click Enable.
