Configure Azure AD SSO

Learn about setting up SSO using Azure AD.


Considerations

  • Setting up SSO/SAML in CloudBolt and Configuring the Identity Provider have been completed.
  • The IDP XML metadata has been captured.
  • At least one Branded Portal has been created for this SSO Provider's use.

Configure Azure AD

In Azure, ensure you have configured an Enterprise Application that is linked to SSO for CloudBolt.

  1. In Azure AD, manage the Enterprise Application.
  2. In the Manage menu, click Single sign-on to access all the data for CloudBolt. The Identifier (Entity ID), Reply URL (Assertion Consumer Service URL) and Logout Url must be set up to point to your CloudBolt instance.


Copy Claim Names for CloudBolt Configuration

User attributes and claims are standard in Azure AD. The defaults can be changed or more added if required.

  1. In User Attributes & Claims, click Edit.
  2. In the Additional claims > Claim name column, copy the Claim names. These will be used during CloudBolt configuration.

  2. Copy the link for App Federation Metadata Url. You will need this for CloudBolt configuration.

Tip: Make sure you have added Users/Groups in CloudBolt to be able to access this Enterprise Application.

Set up Azure AD SSO in CloudBolt

  1. Follow the steps on Setting up SSO/SAML in CloudBolt.
  2. On the Configure a Generic SAML Provider form, complete the fields with information from your Azure Tenancy.
  3. In the Attributes section of the form, copy and paste the attributes from Copy Claim Names for CloudBolt Configuration, step 2.

  1. Click Create.

Upload Azure AD IdP XML

  1. On the Overview tab, click Upload IdP XML.
  2. Choose a source for the Metadata. In this example, Metadata URL is chosen.
  3. ​Paste the App Federation URL copied from Azure AD Identity Provider, Step 2.
  4. Click Save.

Logging in using SSO for Azure AD

Log out of CloudBolt and log back in. You should see a new button called Log in with "SSO Provider Name". If you are logged in with a valid user, you will be redirected and logged in automatically.
