CloudBolt API v3 Authentication

Learn about the authentication requirements to access and execute CloudBolt's API v3.

  • Introduction
  • Prerequisites
  • Token-Based Access to Resources
  • Pass the Bearer Token to Endpoints

Introduction

Authentication to API v3 endpoints requires the use of a Bearer Token in the request header. The API is subject to the same authentication and permissions model as the rest of CloudBolt. This means that the security for which you can perform actions is consistent between the UI and the API.

Requesting the access token requires the same credentials as logging into the CloudBolt web interface.

Prerequisites

  • In order to obtain a Bearer Token, you must have the "API Access" Global Role enabled on your profile.

Token-Based Access to Resources

While the API is available to access for all Users, it cannot be executed until an Authorization Token is acquired.

Authenticate using the Command Line

Use a CLI to authenticate a user. The example below uses a curl command.

  1. Open a CLI tool and submit the following:
    • Enter your CloudBolt instance hostname or IP in the server area.
    • Enter the username and password of the User to generate a Token for.
    • Optionally enter the domain if you are not authorizing a local user.

      Copy Code
      JSON
      curl -X POST\
      -H "Accept: application/json"\
      -H "Content-Type: application/json"\
      "https://<enteryourserverhere>/api/v3/cmp/apiToken"
      '{"username":"xyz","password":"xyz", "domain":"example.com"}'

Authenticate using CloudBolt's API Browser

  1. In your CloudBolt instance, append the API endpoint to your IP. For example: 10.11.12.13/api/v3/cmp/apiToken/
    • Important: You must be logged in to CloudBolt to access the API endpoint.
  2. In the POST Data Content field, delete the token and session field until the structure looks like the following. Optionally enter the domain if you are not authorizing a local user.

    3. Copy Code
    JSON
    {
      "username": "{{username}}",
      "password": "{{password}}",
      "domain": "{{https://your.domain}}"
    }

  3. ​Enter the username and password and click POST.
  4. A unique Bearer Token will return.​

Pass the Bearer Token to Endpoints

Once a Bearer token is obtained, it must be passed to every endpoint via the HTTP request headers. 

Copy Code
Code
{
  "Authorization": "Bearer {{bearer_token}}"
}


Change the Token Default Expiration

The token default expiration is set to five minutes. To change this default, edit the customer_settings.py file and modify the the time (in seconds) to your preference:

Copy Code
Python
import datetime
from settings import JWT_AUTH

JWT_AUTH["JWT_EXPIRATION_DELTA"] = datetime.timedelta(seconds=300)

Error Handling

The API uses standard HTTP status codes to indicate the success or failure of the API call. See HTTP Status Codes for a full list.

The body of the response will be in JSON in the following format:

Copy Code
JSON
{
    "code": 400,
    "errors": [
        {
            "message": "Field 'name' may not be longer than 50 characters"
        }
    ]
}
