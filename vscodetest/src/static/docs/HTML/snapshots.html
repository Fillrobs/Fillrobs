Snapshots

Learn how to use snapshots in CloudBolt to save the state of a virtual machine at its current point in time.


CloudBolt supports managing single-level snapshots on vCenter and Nutanix servers. For single-level snapshots, two actions are provided: creating a new snapshot and reverting to a previous snapshot. Both can be seen on a server’s detail page, as well as scheduling snapshots to occur at designated intervals. 

 For Scheduled Snapshots, see the section below for set up and use details.

All snapshots taken outside of CloudBolt are ignored. CloudBolt does not support reverting to these snapshots nor will CloudBolt ever delete them. 

What is a Snapshot?

Snapshot a server to save the state of the machine at its current point in time. This means marking the state of the server’s hard disk (but not virtual memory). This tool can be very useful when doing development work. If a mistake has been made, or the user otherwise wants to revert their server to the saved state, they can do so by reverting a recent snapshot.

Manually Creating a New Snapshot

Note: This process assumes that scheduled snapshots are not set up.

From the server detail page of a vCenter or Nutanix server, on the left hand side of the page a button called “Create snapshot” will be present. If a snapshot already exists, this button will say "Replace snapshot". It will be there whether the server is currently powered on or off.

  1. To take a new snapshot of the server, click this button.
  2. A dialog will appear asking for two input values. Give the snapshot a name and description (optional).
  3. Submit this dialog to create a new snapshot on the server. This snapshot will be listed in the details tab.

Note that when creating a new snapshot, CloudBolt will automatically delete any existing snapshots that it knows about. This will be done as part of a job. A link to this job will appear as a message at the top of the page once the new snapshot has been created.

Reverting to a Previous Snapshot

To revert a server to a snapshot, a snapshot must first exist on the server that CloudBolt has taken. On the server’s detail page, if a snapshot does exist that CloudBolt knows about, on the left hand side of the page a button called “Revert to snapshot” will be present. To revert the server to the most recent snapshot, click this button. Note that when reverting to a snapshot, the current state of the server’s memory and disk will be lost.

Scheduling Snapshots

Schedule snapshots in CloudBolt for supported servers. Designate a schedule using cron-style scheduling, and set a max number of snapshots. Create manual snapshots at any time outside of the set schedule by using + Create Manual Snapshot.

Note: When enabled, Scheduled Snapshots overrides the manual snapshot creation behavior described above in Manually Creating a Snapshot. 


Setup Overview

  1. Download the Scheduled Snapshots UI Extension from the Content Library.
  2. Navigate to the vCenter or Nutanix server.
  3. Configure the snapshot schedule from the Scheduled Snapshots tab.

Download Scheduled Snapshots UI Extension

  1. Download the Scheduled Snapshots XUI one of two ways: from the Content Library, or from Extensions Management (Admin > System > Manage UI Extensions).
  2. The UI Extension will open in the Extensions Management dashboard.
  • The XUI includes the Take Scheduled Snapshots Recurring Job, which runs every hour and looks for scheduled snapshots to take. This is required for Scheduled Snapshots to function.

Configure Scheduled Snapshots

This functionality requires the Take Scheduled Snapshots Recurring Job. Ensure this Recurring Job is enabled for this feature to function correctly.
  1. In CloudBolt, navigate to Resources > Servers and open the server you would like to schedule snapshots on. This must be a vCenter or Nutanix server.
  2. Click the Scheduled Snapshots tab.
  3. Click the pencil icon next to Schedule.
    1. Enter a schedule using a cron-style string. For example, to schedule a snapshot every hour, every day, enter 0 * * * *. This Cron Calculator is a helpful resource.
    2. Click Save.
  4. Click the pencil icon next to Maximum # of Snapshots to edit.
    • When the maximum number is reached, the oldest snapshot is deleted.
    • This number is set to a default of 3 if left blank.
    • The value set here overrides the default value from the Recurring Job.
