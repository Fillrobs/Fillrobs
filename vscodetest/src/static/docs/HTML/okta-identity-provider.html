Configure Okta SSO

Learn about setting up SSO in CloudBolt using Okta Identity Provider.


Considerations

  • Setting up SSO/SAML in CloudBolt and Configuring the Identity Provider have been completed.
  • The IDP XML metadata has been captured.
  • At least one Branded Portal has been created for this SSO Provider's use.
  • An Okta Administrator account exists that can create an application and add Users.
  • An Okta Organization exists.

Set up Okta in CloudBolt

  1. Log in to CloudBolt as an Admin user.
  2. Click Admin > Security > Single Sign-On (SSO).
  3. Click Add a Single Sign-On IdP > Okta.
  4. Complete the Configure a SAML Provider form.
Note: You will need the Single Sign-On URL and Metadata URL / Entity ID to use in the Okta Application setup below.

Create the Okta CloudBolt Application

In Okta, make sure you are in the Admin menu (https://instance-name.okta.com/admin).

Important: You must be in the Classic UI view in Okta for the following to work. Check your view from the dropdown above the Okta logo.
  1. Click the Applications tab.

  1. Click Add Application > Create New App. Leave "Web" selected and choose SAML 2.0.
  2. In General Settings, enter the App Name.
  3. Click Next to continue to Configure SAML.

Configure Feedback

  1. Select "I'm an Okta customer adding an internal app".
  2. For App Type, select "This is an internal app that we have created".
  3. Click Finish.

Download the IdP Metadata File

  1. In the newly created application in Okta, click the Sign On tab.
  2. In the Settings area, find the SAML 2.0 tip box.
  3. Right click the Identity Provider metadata link > Save Link As > Save it to your local machine.

Assign User Access to the Application

  1. In the newly created application in Okta, click the Assignments tab.
  2. Click Assign > Assign to People.
  3. Search for people or select from the list. Click Assign.
  4. Click Done.

Complete CloudBolt SSO Provider Authentication for Okta

  1. Log in to CloudBolt using a Super Admin user.
  2. Click Admin > Security > See All.
  3. Click Single Sign-On (SSO).
  4. Select Okta.
  5. Import the XML metadata file collected in Download the IdP Metadata File.
  6. Click Edit.
  7. Scroll to the Branded Portal dropdown and select a Portal to use for Okta if one was not chosen at setup time.
  8. (Optional) Enable "Automatically Login to SSO".
    1. Click Admin > Miscellaneous Settings.
    2. Scroll to "Automatically login to SSO" and click Enable.
  9. Sign out of CloudBolt as a Super User. Log back in; you should be redirected to Okta for login.
If successfully configured, a new User account will be created and the user will be prompted to input question/answer for Security question.
