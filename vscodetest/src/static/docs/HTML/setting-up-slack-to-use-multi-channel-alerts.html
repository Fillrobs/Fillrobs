Setting up Slack to use Multi-channel Alerts

Follow this guide to set up Multi-channel Alerts with your company's Slack Workspace. 


Integrating Slack to use CloudBolt's Multi-channel Alerts requires additional steps.

Note: These instructions require using Slack's browser-based application.

Prerequisites

  • Set up or identify the Slack channel to which CloudBolt will send alerts.
  • Log in to your Slack Workspace from a browser or from the application.

Create a Slack App in your Workspace

  1. In your Slack Workspace, click Apps > App directory.
  2. In the top menu bar, click Build.
  3. Click Create an App > From scratch.
  4. Fill in the App Name and select your Slack Workspace.
  5. Click Create App.


  1. In the Add features and functionality section, click Permissions.
  2. Scroll to Scopes > User Token Scopes and click Add an OAuth Scope.
  3. Select chat:write.

  1. A confirmation message will appear. Scroll to the top of this page and click Install to Workspace.


Authorize the App and Copy Access Token

  1. An authorization page will appear after clicking Install App Workspace in step 8 above. Click Allow.
  2. Copy the OAuth Access Token to use in CloudBolt.


Create a Slack Alert Channel in CloudBolt

  1. In CloudBolt, click Admin > Alert Channels > + New Alert Channel.
  2. Select Slack.
  3. In the Edit Slack Alert Channel dialog, enter the following:
    • A general name for your alert 
    • Categories (start with the defaults: “cloudbolt.admin”, “cloudbolt.security”)
    • A channel (# is optional)
    • The Slack API Token you copied from Authorize the App and Copy Access Token, step 2.
  4. Click Save.

Outcomes

Test the integration by logging out and logging back in the CloudBolt. You should see an Alert in your designated Slack channel notifying you of a successful login.
