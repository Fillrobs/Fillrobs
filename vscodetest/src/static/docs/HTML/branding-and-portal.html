Branding and Portals

Branded Portals can be used to customize your CloudBolt portal to fit your company's branding guidelines as presented to users.

Create a Branded Portal

  1. In CloudBolt, click Admin > System > Branding and Portals.
  2. Click
  1. Fill out the Create a Branded Portal Form:
  • Name: Enter a name for this Branded Portal. 
  • Domain/IP: Enter the Domain or IP used to access this CloudBolt portal. The entry here is used for determining the portal and for link generation.
  • Login LDAP Domains: LDAP domains to show on the login page for this portal. This portal is unrestricted if no LDAP domains are selected.
  • Is default: Check to make this instance the default. This will remove the default preference from all other portals.
  • Local Credential Store: Determines whether local user accounts created on this portal are allowed to log in. Enable, disable, or inherit the preference to store credentials locally. Inherit uses the selection chosen in your LDAP Authentication settings.
  1. Click Create

Edit a Branded Portal

Once a Branded Portal has been created, click its Name in the Branded Portal dashboard to begin editing.

Update Colors to New Defaults

Version 9.4.6 of CloudBolt introduced new default colors for the header, footer, and color on hover for the navigation bar links. 

To enable these new colors:

  1. In CloudBolt, click Admin > System > Branding and Portals.
  2. Click the Portal Name to edit.
  3. Click Basic Colors > Reset all colors to defaults. This change will take effect immediately, so ensure you wish to proceed before clicking.
    • Make sure Recalculate advanced colors based on these is checked.
  4. Your Portal will now use the updated CloudBolt default colors.

Edit

Change the general settings for this portal.

  1. From the selected Branded Portal, click
  2. Make any desired changes, then click Save

Images

Add banners, logos, loading image, and login background image for this portal.

  1. From the selected Branded Portal, click Images to browse and add images.
  2. Import the desired images, then click Save

Basic Colors

Customize colors in CloudBolt to match your branding guidelines.
  1. From the selected Branded Portal, click Basic Colors
  2. In each field, enter the color in hex (#000000) format or click the color square to browse.

Advanced Colors

Customize colors for menu hover, security background and text, menu items, and footer background.
  1. From the selected Branded Portal, click  Advanced Colors
  2. In each field, enter the color in hex (#000000) format or click the color square to browse.
  3. Optionally click Derive all advanced colors from basic colors to inherit the colors set in Basic Colors.

Email Settings

Customize the email header image url and email background color for email sent from the selected Branded Portal.

  1. From the selected Branded Portal, click Email Settings
  2. Enter the URL for the Email header image if desired. This image displays in the header of emails sent from the Branded Portal.
  3. In the Email background color field, enter the desired color in hex (#000000) format.
  4. Click

Clone

Copy the Branded Portal and all of its settings.

  1. From the selected Branded Portal, click Clone
  2. Enter a new Name for the portal.
  3. Click Save

Delete a Branded Portal

Delete a branded portal and all its settings. A portal set as Default cannot be removed. Select another portal as default before removing the selected portal.

At least one portal must exist for CloudBolt to display branding. Do not delete all portals from the Branded Portals dashboard.
  1. From the selected Branded Portal, click Remove
  2. Confirm the removal. Click Remove

Determine Active Portals

Each request determines the correct portal to use, first by matching the domain being requested with the 'domain' field on the portals. If no portal is found with a matching domain, the portal with 'default' set to True is used.

The 'name' field of the active portal will be used to replace references to 'CloudBolt' wherever non-admins might see it. Additional image fields are available to override the images used in the header, footer, and the loading icon.

Single Sign-On

Associate an SSO Provider to a Branded Portal to enable the security and ease of Single Sign-On for each portal. This is done from the Single Sign-On Provider configuration.
