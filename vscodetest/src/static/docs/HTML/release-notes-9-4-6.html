Version 9.4.6.2 - Roth


Upgrade Notes

When upgrading CloudBolt, please review all Upgrade Notes between the version of CloudBolt that you are currently running and the target version that you are upgrading to.

Deprecation Warning

  • Several features have been deprecated in this version. See Deprecation Warning for the list.
  • The Physical Server Scan feature will be removed for new CloudBolt users as of 9.4.6 and newer. Existing CloudBolt customers upgrading to 9.4.6 or newer will keep this functionality.

LDAP Domains during Login

  • If you’ve unexpectedly gained additional LDAP Domain(s) as options on the login screen, go to the Domain(s) from Admin > LDAP Authentication Settings and use the Settings button on their details page to disassociate them from the unwanted Branded Portal(s).
  • Alternatively, you could go to the affected Portal(s) from Admin > Branding and Portals and use their Edit button to update the associations from that direction.

Package Updates

  • Oracle SDK upgraded to version 2.33.0.
  • Python has been upgraded to version 3.9.4. If you are using custom code, you may want to test it against Python version 3.9.4.
  • Django has been upgraded to Django LTS 3.2.3. This could impact users who use the deprecated module staticfiles in your XUIs.

Changes to GCPProject id field

  • Moved the text data from the id field to the gcp_id field on the GCPProject model.
  • Standard numerical key added to the id field.
  • GCPHandler.get_env_location returns GCPProject.id data as described above.
  • GCPHandler.get_env_project returns the GCPProject instance instead of GCPProject.id.

If you are using the get_api_wrapper method of a GCP Resource Handler in any of your Actions or other custom code, be advised that you may need to update it so it gets passed the primary key ID of the GCPProject object, rather than the string identifier of the project (e.g., 1 instead of “my-project”).”

9.4.6 New Version Highlights

Introducing Multi-Tenancy in CloudBolt

CloudBolt's Multi-Tenancy capabilities allow a single CloudBolt instance to host multiple distinct customers, who can each administer their own Resource Handlers, Environments, Container Orchestrators, and more, without ever seeing each other's information. Multi-Tenancy in CloudBolt.

UI Refresh

This release introduces many updates to the look and feel of the UI, including:

  • An updated navigation bar style with left and right hand justified buttons.
  • New login page UI with refreshed font, fields, and password toggle icon.
  • Out-of-the-box colors received a makeover for the header and footer, including a hover and on-click color change to indicate which area of the UI you are currently accessing. To enable these new colors, see Branded Portals for steps.
  • Admin Menu updates:
    • Removed "Recent Users" under Security
    • Removed "Recently Added Servers" under Activity
    • Renamed "External Systems" to "Integrations" 
    • Reordered sections on the page
    • Changed "Actions" to "All Actions"
    • Moved "Server", "Resource", and "Orchestration" links to be sub-links under "Actions"
    • Renamed "Django Database Browser" to "Database Browser"
  • Updates to the Catalog page and its Blueprints, Categories, sub-Categories, Filter feature, and Ribbon view. 

GCP Resource Handler

CloudBolt's GCP integration received some TLC this release with added functionality and updates, including:

  • A New OAuth model that requires the use of user/password instead of service account. After upgrading to 9.4.6, Edit the GCP Credentials and provide the new OAuth JSON.
  • The SSL Verification checkbox is now required to be enabled on the Resource Handler form.
  • Import Shared Images across Projects. DEV-17899 
  • Added a Stats tab for GCP Servers. DEV-18424
  • Added support for GCP Labels, which are called Tags in CloudBolt. DEV-19581
  • Added support for passing startup scripts to VMs as Environment Parameters. DEV-19448
  • Added the ability to set custom hostnames on GCP VMs as an Environment Parameter. DEV-18880
  • Introducing a GCP Bigtable Blueprint. DEV-19412
  • Updated the Google Storage Cloud existing Blueprint to support GCP. DEV-16198
  • Create GCP Projects in CloudBolt's UI, using the Create GCP Projects Blueprint. DEV-19401
  • Added the ability to select GCP Disk Type during provisioning and when adding additional Disks to an existing Server. DEV-19472

(Beta) New Terraform Configuration Deployment Item on Blueprints

Terraform Configuration in CloudBolt

This feature is a Beta feature with 9.4.6 and will continue to develop with future versions.

  • Add a Terraform Blueprint Item to a Blueprint from the Build tab. Then link the Terraform Blueprint Item to your .tf config file using one of the following methods:
    • A git repository from a URL.
    • A configuration uploaded as a zip file.
    • A zip file containing configurations from a URL.
    • A configuration directory on the CloudBolt server's filesystem.
  • Map Parameters to the Terraform Blueprint Item that will then appear on the Order form. The default variable dropdown fields are defined in the .tf file.
  • Define and Map custom key:value pairs of Resource Handler variables to Terraform variables. One use case for this is so customers can pass the Resource Handler’s credentials down into the Terraform operating environment.
  • Optionally fetch from remote source on every Terraform Blueprint Item order using Refresh on order.
  • Option to preserve the configuration directory to allow one instance of your Terraform code to be used across multiple Terraform Blueprint Items.
  • Configure one Terraform Blueprint Item across multiple Environments.
  • Override working directory: When Preserve config dir is enabled, the directory entered here will be where CloudBolt will store files used by multiple Blueprint Items. If a user creates a second Blueprint Item, the directory entered here will override the working directory and both Blueprint Items will use the same code.

Resource Types (XaaS)

Improvements to Resource Types are coming with this release, including:

  • All User types can now see the Resources navigation menu item; helper text explains which users are able to access Resources.
  • Added a Resource Icon that can be changed with the Edit Attributes button. Icon shows:
    • Next to the Resource Type in the list of Resource Types.
    • Next the plural Resource Type Name in its list of instances.
    • Above the Actions in the view for a specific Resource instance.
  • Updating the looks of the Action buttons:
    • Include a consistent margin between the Action buttons and the main page content like in a Server management page.
    • Allow for a larger text area within each button by wrapping the text.
  • Updating the content:
    • Show the “Scale” action for Resources that have a Blueprint and hide it for Resources that do not have a Blueprint.

Custom Resource Report

The Custom Resource Report is a new Report in CloudBolt that, when run with specified inputs, returns a list of Resources by Resource Type. Select one or multiple Resource Types to be included in the Report results. Choose which columns display in the results table.

CloudBolt Cost Management UI Extension

Use the Cloud Cost Management (Kumolus) UI Extension to keep track of your Resource Handler's spend and efficiency within CloudBolt's UI.
