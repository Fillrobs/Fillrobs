Grant Users Access to OneFuse

Learn how to enable existing CMP Users to access and use OneFuse.


  • Prerequisites
  • Add User(s) to the Group for OneFuse
  • Add the API Access Global Role to the User
  • Access OneFuse from CMP

Prerequisites

  1. License OneFuse
  2. Identify User(s) to access OneFuse

Add User(s) to the Group for OneFuse

  1. Log in to CMP UI with Admin permissions to manage Users & Groups.
  2. Click Admin > Groups (under Security)
  3. Click on the Group named Default
    • The page's breadcrumb will show: Groups / Workspace: Default
  4. In the Users tab, click on the button to add User(s)
  5. Add Users to Group Default form:
    • Roles to grant:
      • Add only one (1) out of the three (3) Roles:
        Note: By default, the Role Viewer is selected. Remove the Viewer Role if the User should be a Workspace Admin or Workspace Member.
        • Workspace Admin
        • Workspace Member
        • Workspace Viewer
          • In the event Workspace Viewer is not available from the list, select Viewer
        Note: If any other Roles are added for the User in the Group, the User will not be able to access OneFuse properly.
    • Select all desired Users for that Role in the table
    • Click Add Users
  6. Repeat Step 4 through Step 5 until all desired Users with the appropriate OneFuse Role are added to the Group.

Refer to Groups, Roles and Permissions and Permissions for more information.

Add the API Access Global Role to the User

  1. Click on Admin > Users
  2. Find the User from the list
  3. Click on the User's email link to edit the User
  4. Click on Manage profile
  5. For the Manage profile form:
    1. Check the checkbox for API Access
    2. Click Save changes
  6. Click on the Users link from the page breadcrumb to go back to the list of all Users
  7. Repeat Step 2 through Step 5 until all Users that need OneFuse access have the API Access Global Role

Access OneFuse from CMP

After the User is added to the Group for OneFuse, has the necessary Group Role and has the Global Role: API Access, the User will be able to access OneFuse.

  • The User should be able to access OneFuse UI via the browser by appending /ui to the CMP instance URL
    • For example, if the instance URL for CMP is: https://cmp.example.com, access OneFuse UI via https://cmp.example.com/ui
  • Optionally, when the User is signed in, the User can click on their Profile icon from the top navigation bar and see Go to OneFuse
