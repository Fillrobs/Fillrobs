Configure Office 365 SSO

Learn about setting up SSO using Office 365.

Note: Office 365 uses the same setup as AzureAD. The screenshots in this setup are from the AzureAD SSO interface.

Considerations

  • Setting up SSO/SAML in CloudBolt and Configuring the Identity Provider have been completed.
  • The IDP XML metadata has been captured.
  • At least one Branded Portal has been created for this SSO Provider's use.

Configure Office 365

In Office 365, ensure you have configured an Enterprise Application that is linked to SSO for CloudBolt.

  1. In Office 365, manage the Enterprise Application.
  2. In the Manage menu, click Single sign-on to access all the data for CloudBolt. The Identifier (Entity ID), Reply URL (Assertion Consumer Service URL) and Logout Url must be set up to point to your CloudBolt instance.

Copy Claim Names for CloudBolt Configuration

User attributes and claims are standard in Office 365. The defaults can be changed or more added if required.

  1. In User Attributes & Claims, click Edit.
  2. In the Additional claims > Claim name column, copy the Claim names. These will be used during CloudBolt configuration.

Set up Signing and Notification Email

  1. In this example, Sign SAML Assertion and SHA-256 are set, as well as an email address for notifications.
  2. Gather the Federation Metadata. You will need this for CloudBolt configuration.
    • Either copy the App Federation Metadata Url or Download the Federation Metadata XML file.
  1. In Office 365, configure Step 4 as shown below.

Step 5 in Office 365 is an option to Test Single Sign On with App Name. This will not work until SSO is configured in CloudBolt.

Tip: Make sure you have added Users/Groups in CloudBolt to be able to access this Enterprise Application.

Set up Office 365 SSO in CloudBolt

  1. Follow the steps on Setting up SSO/SAML in CloudBolt.
  2. On the Configure a Generic SAML Provider form, complete the fields with information from your Office 365 Tenancy.
    1. On the Single Sign On configuration, set: Name ID Format* to Email address, and uncheck "Response Signed".
  3. In the Attributes section of the form, copy and paste the attributes from Copy Claim Names for CloudBolt Configuration, step 2.
  4. Click Create.

Upload Office 365 IdP XML

  1. On the Overview tab, click Upload IdP XML.
  2. Choose a source for the Metadata. In this example, Metadata URL is chosen.
  3. ​Paste the App Federation URL copied from Office 365 Identity Provider, Step 2.
  4. Click Save.

Logging in using SSO for Office 365

If User account creation is successful, CloudBolt will prompt the User to create a Security Question/Answer.

Log out of CloudBolt and log back in. You should see a new button called Log in with "SSO Provider Name". 

  • If you are logged in with a valid user, you will be redirected and logged in automatically.
  • If you are logged out of Office365, this will send a 2 factor auth request if it is working as expected.
