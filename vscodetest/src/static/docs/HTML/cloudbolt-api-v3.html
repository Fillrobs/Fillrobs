Overview

CloudBolt's API is one way to get information into and out of the CloudBolt Platform. Use the CloudBolt API to automate your processes and increase efficiency when accessing information in CloudBolt from your third party application.

Perform a wide variety of tasks using API v3, such as creating Environments, Groups, and Resource Handlers, creating, deleting, and managing Servers, updating Roles, and more.

API v3 follows the RESTful methodology. Our API:

  • Has predictable resource-oriented URLs.
  • Accepts JSON request bodies. Some endpoints also accept form-encoded request bodies and multipart form-data for file uploads (importing Blueprints, for example).
  • Returns HAL JSON-encoded responses.
  • Uses standard HTTP response codes, authentication, and verbs.
For best results, we recommend using an API editor like Swagger UI or Postman.

Accessing the API

CloudBolt's API Browser

CloudBolt offers an API browser with endpoints, available verbs, and the ability to Execute the endpoint with a returned response. Only Users with the "API Access" Global Role can execute the API. See API Authentication.

  1. In CloudBolt, click Admin > System > API Browser.
  2. Click the API Console  icon.
  3. Scroll to the beginning of the API v3 endpoints and select the desired endpoint(s) for more actions.

API Editors

Use an API Editor like Swagger to view, search, authenticate, and execute CloudBolt's API v3. See Swagger User Interface.

Methods

CloudBolt API v3 uses a RESTful approach and is comprised of resources (collections) and verbs (or methods).

Methods can be used to interact with an entire resource list:

  • /environments

A specific item by ID:

  • /environments/{id}/

Or specific information about a specific item:

  • /gcp/projects/{id}/networks/{network_id}


Available Methods

The following methods are available using CloudBolt's API v3:

Fetch a resource / collection and return the value(s).
Create a resource / collection and return the created value(s).
Replace a resource and return the new value.
Update a resource and return the updated value.
Delete a resource.
Not all methods are available for every endpoint.

Parameters

The CloudBolt API uses {id} or {resourceID} as parameters to return specific items from a collection.

  • This value can be the ID assigned to resources in CloudBolt, or specific items from a remote source such as {networkID}, etc. 

Find the ID for a Resource

Interact with objects using a unique Global ID value. 

  1. Using the API, request a list of a resource using GET.
  2. The response will list the resources, objects related to that resource, and the ID.

The following example shows a request to list Resource Handlers. 

  • For the _embedded property in the response, find the list of individual Resource Handlers, their titles, and Global ID.
  • Each item in that list has a _links section that allows navigation to the objects related to the resource handler as well as a "self" link that provides the full URL to access the individual Resource Handler's data.
  • Beneath each Resource Handler's _links section is the specific data for that resource handler, including id and title.

The first _embedded property is /api/v3/resourceHandlers/vmware/RH-d6t7k254/. The last string is the Global ID.

Error Handling

Below is a table containing all of the error codes currently utilized and a short description of their cause. To handle an error, you can check the description for the returned status code and modify your call accordingly.

HTTP
Error Name
Description
Client-Side Errors
400
INVALID_ARGUMENT
Client specified an invalid argument. Check error message and error details for more information.
400 FAILED_PRECONDITION Request can not be executed in the current system state, such as deleting a non-empty directory.
400 OUT_OF_RANGE Client specified an invalid range.
401 UNAUTHENTICATED Request not authenticated due to missing, invalid, or expired OAuth Token.
403 PERMISSION_DENIED Client does not have sufficient permission. This can happen because the OAuth token does not have the right scopes, the client doesn't have permission, or the API has not been enabled for the client project.
404 NOT_FOUND A specified resource is not found, or the request is rejected by undisclosed reasons, such as whitelisting.
409 ABORTED Concurrency conflict, such as read-modify-write conflict.
409
ALREADY_EXISTS
The resource that the client tried to create already exists.

Either out of resource quota or reaching rate limiting. Look for "Cloudbolt.rpc.QuotaFailure" error detail for more information.
499
