VMware vRealize Orchestrator (vRO)

Learn how to set up and use VMware vRealize Orchestrator in CloudBolt Software.


Use VMware vRealize Orchestrator in CloudBolt to Import Orchestration Flows and execute them from Blueprints. Inputs defined in the flows will be available on the Order form.

Add VMware vRealize Orchestrator

  1. In CloudBolt, navigate to Admin > Orchestration > External Orchestrators.
  2. Click + Add an Orchestration Engine
  3. ​Choose VCenter/vRealize Orchestrator.
  4. Fill in the required fields:
    5. Scheme Protocol used to connect to this orchestration engine.
    Host IP address/hostname for your VMware vRealize Orchestration instance.
    Port  Port used to connect to this VMware vRealize Orchestration instance.
    Account Username used to run workflows on this VMware vRealize Orchestration instance.
    Password Password for the VMware vRealize Orchestration account.
  5. Click Continue
  6. ​Enter a Name for the Orchestrator and a Base Folder for discoverable workflows.
    7. Name Enter a name for the Orchestrator that will appear in CloudBolt.
    Base folder for discoverable flows This folder allows importing Orchestration Flows from the specified folder. The folder entered here must contain workflows; it will not detect sub-folders.
  7. Click Done

Edit VMware vRealize Orchestrator

  1. On the External Orchestrators dashboard, click the Name of your Orchestrator.
  • Edit Credentials: Change the credentials used to authenticate the connection.
  • Edit Settings: Change the name of the VMware vRealize Orchestrator and base folder.
  • Verify Connection: Click to verify the connection to VMware vRealize Orchestrator.
  • Delete: Click the remove this Orchestration Engine. Click Remove to confirm.

Import VMware vRealize Orchestrator Flows

  1. On the External Orchestrators dashboard, click the Name of your Orchestrator.
  2. Click Flows.
  3. Click Import Flows

Execute Blueprints using Flows

Once a VMware vRealize Orchestrator connection is established and workflows are imported, set up your Blueprint to use the imported flows.

  1. In CloudBolt, navigate to the Catalog.
  2. Create a new Blueprint or open an existing Blueprint.
  3. Click Build.
  4. Click + Action.
  5. Click External Flow.
  6. Click + Create a new flow action now.
  7. Assign a name to your new flow, then scroll to the Flow dropdown and select the target flow.
  8. Click Save.

The Flow will now be triggered when you submit an order using this Blueprint.
