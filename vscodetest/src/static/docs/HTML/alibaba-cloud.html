Alibaba Cloud

Learn how to set up the Alibaba Cloud Resource Handler in CloudBolt to provision and manage servers in your Alibaba Cloud instance.

Configuring Alibaba Cloud

Complete the following prerequisites before setting up the Alibaba Cloud Resource handler in CloudBolt

  1. Obtain an Access Key and Access Key Secret.
    • Follow the steps below to obtain a Access Key/Access Key Secret:
      1. Log in at https://usercenter.console.aliyun.com
      2. Click Create AccessKey.
      3. Click Save AccessKey Information.
      4. A CSV file will be downloaded that contains both the key and its secrets.
  2. Determine your Region.

Setting up the Alibaba Cloud Resource Handler

  1. Log in to CloudBolt UI
  2. Navigate to Admin > Resource Handlers.
  3. Click +Add a resource handler...
  4. Select Alibaba Cloud.
  5. Enter the Region, Access Key, and Access Key Secret obtained from the section above. 
  6. Click Continue
  7. Enter a Name and an optional Description
  8. Click Done

You will be directed to the newly created Alibaba Cloud Resource Handler where you can:

  1. Import Images - at least one Image is required
  2. Import Networks - at least one Network is required
  3. Add and Manage Tags - optional
  4. Configure the Environment that was auto-created for this Alibaba Cloud Resource Handler
Additionally, view a list of Servers, Compute Limits and History once Orders exist.

1. Import Images

Import at least one Image that will be available on the Order Form for this Resource Handler.

  1. On the desired Alibaba Cloud Resource Handler, click the Images tab
  2. Click the Import Images button
  3. Select at least one Image from the Discover Images list
  4. Click Import selected Images

Once at least one Image exists, manage the Images' OS Build Environments and Credentials.

2. Import Networks

Import at least one Network that will be available on the Order Form for this Resource Handler.

  1. On the desired Alibaba Cloud Resource Handler, click the Networks tab
  2. Click Import networks button
  3. Select at least one Network from the Import Networks list
  4. Click Import selected networks

3. Add and Manage Tags (Optional)

CloudBolt creates and tags resources as part of the provisioning process. Optionally add Tags for that are linked to the CloudBolt server attributes for Alibaba resources.

  1. On the desired Alibaba Cloud Resource Handler, click the Tags tab
  2. Click the Add Tag button
  3. Fill out the Add Tag form:
    Field Name Comments
    attribute Select from the dropdown list on the attribute that will provide the value for the new Tag
    Tag Name Specify the new Tag's name
  4. Click Add
  5. Once the new Tag is added to the list, the Syncing behavior (Syncing column) is displayed:
    • When a Tag's syncing is marked as Bidirectional, the Tag's value will update in Alibaba Cloud and CloudBolt if changed in either location.
    • When a Tag's syncing is marked as Out Only, the Tag's value is will update in Alibaba Cloud from CloudBolt.
    • When a Tag's attribute is group or owner, you will be able to toggle the Sync behavior.

Manage existing Tags by editing the Tag's attribute and/or name and delete unnecessary Tags.

Tag Behaviors

  • Tags are synced as part of the Refresh Info process on a server
  • Tags are initially imported on an initial VM discovery, or updated if the Tags already exist in CloudBolt when running Sync VMs
  • Tags are set during provisioning when configured with attributes on the order
  • Tags are updated in the Resource Handler when updating the associated attribute in CloudBolt (e.g. editing the value for a Parameter on the server)
  • Removing a tag mapping does not effect the current stage of Tags in the Resource Handler, including the server
Learn more about Tags.

4. Configure the Environment

Configure the Alibaba Cloud Environment that was auto-created for this Alibaba Cloud Resource Handler before ordering servers.

  1. On the desired Alibaba Cloud Resource Handler, click the Overview tab
  2. Find the Environments row and click on the Environment link
  3. You will be re-directed to the Environment's settings page where you will be able to:
    1. Modify Alibaba Cloud Parameters - Optional
    2. Add CloudBolt Parameters and Setup User-Data Parameters - Optional
    3. Assign Groups - Required
      1. Specify the Groups for this Alibaba Cloud Environment by clicking on Groups tab
      2. Click on the Edit button
      3. Select all the Groups desired
      4. Click Save
    4. Add OS Build(s) - Required
      1. Specify the OS Build(s) for this Alibaba Cloud Environment by clicking on OS Builds tab
      2. Click on the Edit button
      3. Select all the OS Build(s) desired
      4. Click Save
    5. Add Network(s) - Required
      1. Specify the Network(s) for this Alibaba Cloud Environment by clicking on Networks tab
      2. Click on the Make changes button
      3. Click Add a network
      4. Select a network from the menu and specify desired NIC(s)
      5. Repeat Step 3.e.iii and Step 3.e.iv until all desired network(s) are added
      6. Click Save changes
    6. Set Limits - Optional

Alibaba Cloud Parameters

The following Alibaba Cloud Parameters are supported.

Parameter Comments
Alibaba Disc Category
For orders, the selection dropdown will update based on the Zone and Instance Type selected.
Select from a list of existing Disc Categories for the Alibaba Region.
Allocate Public IP? When set to true, the Internet Max Bandwidth Out is required.
Instance Type
For orders, the selection dropdown will update based on the Zone selected.
Select from a list of existing Instance Types for the Alibaba Region.
Internet Max Bandwidth Out
This Parameter is required when Allocate Public IP? is set to true. The value should be an integer greater than or equal to 1.
Alibaba Key Pair Select from a list of existing SSH Keys for the Alibaba Region.
Alibaba Credential Type Select either Key Pair or Password.
Alibaba Login Password Required. Specify the Login Password.
Alibaba Resource Group The Alibaba Resource Group to provision into.
Security Group
For orders, the selection dropdown will update based on the Network selected.
Select from a list of existing Security Groups for the Alibaba Region.
Security Hardening When set to true, CloudBolt sends the request to Alibaba with the Enable Security Hardening flag. Alibaba will perform Security Hardening for images that support this functionality.
This feature is enabled at provision time only and cannot be enabled after the initial deployment.
Zone Select from a list of existing Zones for the Alibaba Region.
