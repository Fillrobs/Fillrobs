Recurring Jobs with Secrets

When a recurring job with type Action is selected for export, any secrets (values for Action type recurring jobs with the type 'password' or 'encrypted text') can be encrypted with a user-designated password. When the Action type recurring job is imported, another prompt will appear to enter the user-designated password. This decrypts secrets when the recurring job is imported.

This is useful for maintaining security when sharing exported objects across different CloudBolt instances.

If a user-designated password is not entered in the prompt, a default key will be used. In this case, the exported recurring job's secrets can only be decrypted when imported to the same CloudBolt instance it was exported from.

When exporting/importing through the API, secrets are encrypted/decrypted with the default key only. We recommend using the CloudBolt UI to provide a user-designated password.
