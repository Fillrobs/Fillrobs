Connection Info

Learn how to use Connection Info objects in CloudBolt.

Introduction

Connection Info objects are used to securely store credentials used for connections to systems outside of CloudBolt. Each Connection Info object can contain IP/Hostname, port, username, password, and more. There is a standard Connection Info configuration and another for Git Connection Info objects. Any custom action can access the securely stored credentials in a Connection Info object.

Connection Info for Remote Source

Connection Info objects used for Remote Source (Blueprints, Actions, etc.), must be named "Source Code Repository" in the Connection Info form.

Add Connection Info

  1. In CloudBolt, click Admin > Integrations > Connection Info.
  2. Click + New Connection Info
  3. Fill out the Create new Connection Info form.
Field name Notes
Name A descriptive name for this connection.
IP/Hostname Either the IP or the Hostname for this connection.
Port The Port used for this connection.
Protocol HTTP or HTTPS are the standard options. See Development Tips for Plug-ins for more options.
Username The username of the account being connected to.
Password The password of the account being connected to.
SSH Key An SSH Key used in place of a password. If an SSH Key is selected and a script is run on this connection with SSH, any password entered will be ignored in favor of using the chosen key.
Use Auth Headers Check to enable the use of Headers.
Headers

Enter Header text here in JSON format. Used as a replacement for username:password configurations.

Example: 

Copy Code
JSON
{'Content-Type': 'application/json', 'Accept': 'application/json'}
Labels Labels are a way to organize and filter Connection Info entries. Begin typing in the Labels field to add a new Label or choose an existing Label.

Add Git Connection Info

This is useful for Git-hosted Terraform Plan Connection information.

  1. In CloudBolt, click Admin > Integrations > Connection Info.
  2. Click + New Git Connection Info
  3. Fill out the Create new Connection Info form.
Field name Notes
Name Enter Source Code Repository in this field for Git Connection Info.
Host The host for the source control connection in domain format (github.com).        
Protocol Choose HTTP or HTTPS.
Username The username of the source control account.
Auth Token If the source control account uses auth tokens instead of passwords, enter it here. If no auth token is used, enter a password here.
Labels Labels are a way to organize and filter Connection Info entries. Begin typing in the Labels field to add a new Label or choose an existing Label.
If multiple Connection Infos with the same host have the terraform-plan-source label, the protocol field is used to filter further. If multiple Connection Infos are still found, CloudBolt chooses the first one based on Connection Info ID.
This feature only supports HTTPs and HTTP for repositories requiring authentication. (e.g., https://github.com/myco/my-terraform-plans.git)This feature does not support SSH git sources. (e.g., git@github.com:myco/terraform-plans.git)

Edit Connection Info

Edit Connection Info post-creation using the [] Edit action.

  1. In the Connection Info dashboard, find the Name of the Connection.
  2. In the Actions column, click the Edit [] icon.
Take caution editing the name of a Connection Info entry. Some functionality depends on looking up a specific name based on the entry here.

Editing is also available on the details page of a Connection Info entry.

Delete Connection Info

Deleting Connection Info is an irreversible action.

  1. In the Connection Info dashboard, find the Name of the Connection.
  2. In the Actions Column, click the X.
  3. Click Delete to confirm.

Advanced Connection Info Configuration

Connection Info objects can be used in Plug-Ins to securely store credentials, or to run scripts on legacy technology that may not have a REST API. See the example below for using Stored Credentials, or visit Development Tips for Plug-ins for more examples of Connection Info objects in Plug-ins.

Storing Credentials in Plug-ins

  1. Create a Connection Info object with an easily identifiable name.
  2. In your Plug-in, access and make use of the Connection info object using code like the following example.
Copy Code
Python
from utilities.models import ConnectionInfo
connection_info = ConnectionInfo.objects.get(name='my_connection_info')
ip = connection_info.ip
username = connection_info.username
password = connection_info.password
