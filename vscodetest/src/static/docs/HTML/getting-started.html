Getting Started with CloudBolt

Now that you've installed CloudBolt, complete the following basic tasks to connect to your virtualization platform and begin provisioning servers.

Quick Setup

  1. Log in for the first time.
  2. Add a Resource Handler, import Templates and Networks.
  3. Configure the Environment.
  4. Order Server(s).

Log in for the First Time

Log in to CloudBolt using either the Admin User you set up during Guided Setup, or using the default credentials admin / admin if you skipped Guided Setup.

Create Users, at least one of whom is a Super Admin, and decide if your organization needs to utilize Legal Notice Text.

Add Resource Handlers

A CloudBolt Resource Handler is a connection to your virtualization platform or public cloud provider. You will need to add a Resource Handler for each platform and account.

  1. Navigate to Admin > Resource Handlers.
  2. Click + Add a Resource Handler...
  3. Select your provider and complete set up.

Configure the Environments

CloudBolt Environments belong to a Resource Handler and represent a logical grouping of resources within that platform or provider.

The name of the tab depends on the Resource Handler. For Azure and AWS, it is Regions. For vCenter, it is Clusters.


Add Environments

For Azure, AWS and other Resource Handlers, you will need to add Environments by going to the technology specific tab in the Resource Handler. This may be Regions, Clusters, or Locations depending on the technology type.

Configure each Environment

Assign Networks available to the environment

  1. Navigate to the Networks tab.
  2. Select Make Changes.
  3. Select + Add a Network. 
  4. Select a network to add to the Environment and click Add.

Assign Environment to CloudBolt Groups

  1. Either Ensure Groups are unconstrained.
  2. Add at least one OS Build to the Environment.

Order Server(s)

  1. Click Catalog > Order. Fill in the required fields and click Order.
    • Select a Group.
    • Select an Environment.

See Servers.
