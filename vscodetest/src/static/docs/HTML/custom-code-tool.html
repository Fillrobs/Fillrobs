Custom Code Analysis Tool (CCAT)

Learn about the Custom Code Analysis Tool (CCAT), how to use it when considering an upgrade, and view sample output.

Introduction

The Custom Code Analysis Tool (CCAT) is a feature designed to evaluate custom code in a CloudBolt database prior to an upgrade.

The downloaded file is named cloudbolt-upgrade-analysis-tool. Learn how to Install and Run the Tool below.

This tool is available to download from the CloudBolt Downloads Portal. It is also packaged with the 9.4.5+ upgrader.

How it Works

The CCAT scans for custom code in the current or user-designated CloudBolt version in specified areas (listed below). Then, the tool checks signatures of every function, class, method, model, and field in the most recent version of CloudBolt’s code (the target version).

  • The tool provides a report of the areas only where you have written custom code that have changed in the most recent version.
  • Areas that are scanned include Actions, configuration files that support Container Object Build Items on Blueprints, and Python files in UI Extensions.

You may need to address those areas of your custom code before upgrading, because your integrations may not work the same way in the target version.

The current version can be overridden using the --current flag. The target version by default is the latest version of CloudBolt supported by the tool.

Goal

Collect information on the areas of your CloudBolt's custom code that may be affected during an upgrade. Deliver the report to the party in your company who is responsible for the custom code to make any necessary changes to prevent issues during upgrade.

Prerequisites

  • The IP address of the CloudBolt instance you wish to test.

Concepts

Snapshot

In the context of the CCAT, a Snapshot is a static image of a CloudBolt repository by version. The CCAT includes snapshots of all currently supported versions of CloudBolt:
  • v8.8
  • v9.0
  • v9.1
  • v9.2 and its minor versions
  • v9.3 and its minor versions
  • v9.4 and its minor versions

Flag

A Flag in CCAT is a Python syntax that allows you to request certain information from the tool. Available flags are:

  • --help: The --help flag displays introductory text in your command-line interface and lists the available flags.
  • --version: Show the version of the CCAT.
  • --c, current: Represents the current version of CloudBolt you are upgrading from.
  • --t, target: Represents the target version of CloudBolt you are upgrading to.

Install and Run the Custom Code Analysis Tool

Follow the instructions below to install and run the CCAT on your CloudBolt instance. The CCAT is also available to download from the CloudBolt Downloads Portal.

  1. Using a command-line interface, SSH into your CloudBolt instance: ssh root@<your.cloudbolt.ip>
  2. Run the following commands in order:
    1. curl https://downloads.cloudbolt.io/ccat/cloudbolt-upgrade-analysis-tool -o /usr/local/bin/cloudbolt-upgrade-analysis-tool
    2. Once you untar the upgrader, cd to the untar directory and change the file permissions to: chmod +x /usr/local/bin/cloudbolt-upgrade-analysis-tool
    3. cloudbolt-upgrade-analysis-tool --help

Run the Tool

After installation is complete, use the command cloudbolt-upgrade-analysis-tool paired with a flag to use the CCAT. In the example output below, the following command is used:
