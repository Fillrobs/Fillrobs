Upgrade Using a Yum Repository

To help make upgrades more reliable, we have removed operating-system packages (RPMs) from the Installer and Upgrader. The Upgrader will now update all system requirements to the latest versions available.

Users with Public Internet Access

Before you Begin

Users who would like to upgrade should verify that their appliances have a Yum repository available. 

  1. SSH into your appliance.
  2. Run yum check-update

If your appliances are hosted in firewalled environments, you may need to add proxy settings to those appliances' yum config:

  1. In the yum config file /etc/yum/conf, define the proxy settings to the following:

    Copy Code
    Code
    [main]
    ………………
    proxy=http://<Proxy-Server-IP-Address>:<Proxy_Port>
    proxy_username=<Proxy-User-Name>
    proxy_password=<Proxy-Password> 
    ………………
     
  2. Save and exit the file. Start using the yum command. 
  3. Run a command like the following to check if you are able to fetch the packages: [root@yourfolder ~]# yum-repolist

Users without Public Internet Access

Users who are not able to access a public Yum repository have two options for upgrading their appliances:

  1. Host an internal Yum repository mirror. See Centos docs for instructions.
  2. Upgrade by installing a new CloudBolt Appliance OVA. See Upgrading to 9.X.
In the Upgrading to 9.X instructions, references to 8.X will apply to any version you're currently using.
