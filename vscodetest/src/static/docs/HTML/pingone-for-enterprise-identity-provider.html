PingOne for Enterprise Identity Provider

Learn how to set up SSO in CloudBolt using the PingOne for Enterprise IdP.

These instructions outline the configuration needed to set up PingOne for Enterprise SSO in CloudBolt, as well as account/application configuration in PingOne.

Considerations

  • These instructions requires an administrator account in PingOne for Enterprise to create an application and add Users.
  • These instructions assume a PingOne Organization already exists.
Note: PingOne provides a similar product called PingIdentity SSO. CloudBolt can integrate with this product as well, but this guide is specifically around the PingOne for Enterprise configuration.

Set up PingOne in CloudBolt

  1. In CloudBolt, log in as Admin. 
  2. Click Admin > Security > Single Sign-On (SSO).
  3. Click Add a Single Sign-On IdP > Generic SAML Provider.
  4. Complete the Configure a SAML Provider form.
  5. Click Save.
  6. Copy and store the Single Sign-On URL, Metadata URL/Entity ID, and click Download Metadata XML as these will be used later during PingOne setup.

Create the PingOne for Enterprise CloudBolt Application

Connect to PingOne Directory

  1. Log in to the PingOne for Enterprise web interface.
  2. In the top navigation menu, click Setup> Identity Repository > Connect to an Identity Repository.
    1. Connect to an Identity Repository: Select PINGONE DIRECTORY > Next.
    2. ​After the directory has been configured, click Next.
    3. Make note of the Map Attributes, which will be used later.
  3. Click Finished.

Add PingOne SAML Application

  1. In the top navigation menu, click Applications > Add Application > Search Application Catalog.
  2. Search for "Ping" in the Search field, then select the arrow next to Ping IDaaS Directory Provisioner > Setup.

1: SSO Instructions

  1. Download the Signing Certificate.
  2. Copy and store the IDs, URL, and Issuer for later use.
  3. Click Continue to Next Step.

2: Connection Configuration

  1. Upload the Metadata URL you collected from CloudBolt during Set up PingOne in CloudBolt, step 6.
    • The ACS URL and Entity ID fields will populate based on the Metadata URL file. These fields should match your SSO Provider in CloudBolt.
  2. Click Continue to Next Step.

3: PingOne App Customization

  1. This section is optional; change the PingOne App icon, Name, Description, and Category if desired.
  2. Click Continue to Next Step.

4: Group Access

  1. If you have Groups, add them here.
  2. Click Continue to Next Step.

5: Review Setup

  1. Review all of the presented information.
  2. Copy and store the Initiate Single Sign-On (SSO) URL. You will need this to test the SSO connection later.
  3. Download the Signing Certificate and SAML Metadata.
  4. Click Finish.

Upload Metadata to CloudBolt's PingOne for Enterprise SSO Configuration

Navigate back to your CloudBolt PingOne SSO Configuration to upload XML Metadata.

  1. In your PingOne SSO Configuration, click Upload IdP XML.
    • Metadata Source: Metadata file
    • Metadata File: Select the file you downloaded from 5: Review Step, step 3.

  2. Upload the Certificate if desired.

Test the PingOne for Enterprise SSO Connection

  1. In CloudBolt, click Admin >Users and create a new User. 
    • Make sure the new User's Group membership matches any Group entered in PingOne setup.
  2. Navigate to the Initiate Single Sign-On (SSO) URL and log in with the new User account.
  3. If you are redirected to CloudBolt and you are able to log in, congratulations! The SSO configuration is complete.

Configure Branding

  1. In CloudBolt, click Admin > System > Branding and Portals.
  2. Click Add a portal on the top left.
  3. In the Add a portal window, fill out your Name, Domain/IP and select your Single Sign-On Provider.
  4. Click Create.
  5. Log out of the CloudBolt appliance and log back in.
  6. Now you should see a button at the bottom saying Log in with [Name entered in Step 3].
  7. Click Log in with [Name entered in Step 3] and login with your credentials.
  8. If your account is valid, you will be logged into the CloudBolt appliance.
