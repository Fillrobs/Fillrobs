Deploy in AWS

Learn how to deploy the CloudBolt Appliance in AWS.

There are two ways to start the AMI based deploy of your CloudBolt Appliance:

  • Option 1: ​CloudBolt AMI from the public image
  • Option 2: CloudBolt AMI from the AWS Marketplace
Important: These are "Bring Your Own License" instructions. If you need a CloudBolt license, please contact sales@cloudbolt.io.

Option 1: ​CloudBolt AMI from the Public Image

A wider selection of CloudBolt AMI builds may be available as public images from CloudBolt Software. If you do not require a specific version, continue to Get CloudBolt from the AWS Marketplace.

  1. Navigate to the CloudBolt AWS AMI of choice:

    2. CloudBolt Version Region AMI Link
    9.4.4 us-east-1 CloudBolt AWS AMI (us-east-1)

  2. Click Launch and proceed to Configure the CloudBolt appliance.

Option 2: CloudBolt AMI from the AWS Marketplace

  1. Login to AWS Marketplace.
  2. From the left-hand menu, click Discover products.
  3. Find CloudBolt.
  4. Click Continue to Subscribe
  5. Click Accept Terms to subscribe then wait for the subscription process to complete. If you already have a subscription, continue to Configuration.
  6. Click Continue to Configuration
  7. On the Configure this software page, choose:
    1. Delivery Method: “64-bit (x86) Amazon Machine Image”
    2. Software Version: The latest available version of CloudBolt 9.x
    3. Region: Select the region you would like to launch your instance. The default is US East (N. Virginia).
  8. Click Continue to Launch
  9. Choose Action: Launch through EC2.
  10. Click Launch and proceed to Configure the CloudBolt Appliance.

Configure the CloudBolt appliance

Choose Instance Type

  1. Select the AWS EC2 Instance Type you would like to use for your newly launched instance. For help choosing your instance type, see CloudBolt's Hardware Guidelines or choose m4.large to get started.
  2. Click Next: Configure Instance Details

Configure Instance Details

  1. Network: Select the VPC network or AWS EC2 network you would like to launch the instance in.
  2. Select the Subnet. To visit CloudBolt from your browser, the subnet you connect to must have port:443 available to the network you are connected to.
  3. Click Next: Add Storage


Add Storage

  1. Configure your root storage:
    1. Size: 50GB for a minimal install, increase this to 100GB for a production installation.
    2. Encryption: Recommended. Choose (default) or the appropriate KMS Key. Consult your AWS Admin if you do not know the appropriate key for your account. 
    3. Volume Type: GP2 or GP3
  2. Click Next: Add Tags

Add Tags

Tags are a way to group instances, volumes, or both for Billing. We recommend you at least add a Name tag to identify your CloudBolt instance along with any others required in your environment.

Tagging is one way to filter your instances after launch, which can make it easier to find during initialization. 

Security Group

  1. Select Create a new security group.
  2. Change the Security group name to an identifiable label.
  3. Change the Description to an identifiable label.
  4. Select the security group you would like to use for this instance. 
    1. In the Source dropdown, select My IP or ask your Network Admin for guidance.
    2. ​Click Add Rule and follow the input for HTTPS from the table below.
    3. If you are using a custom security group, please ensure that all of the ports are correctly listed so the security group can grant access appropriately.
    • Type Protocol Port Range Source Description
      SSH TCP 22

      My IP or recommendation from Network Admin

      • Used to remotely administer your appliance.
      • It is recommended that you restrict this port to trusted IP addresses.
      • If you do not want to do this, leave the source as 0.0.0.0/0.
      		CloudBolt Admin access. Not Required for end users.

      		End User access to CloudBolt

  5. Click Review and Launch
  6. Review your Details, then click Launch

Add a Key Pair

A key pair consists of a public key that AWS stores and a private key file that you store. Together, they allow you to connect to your instance securely. The private key file allows you to securely SSH into your CloudBolt instance. Keep it safe, as it may be required to perform maintenance on your appliance in future.
  1. In the dropdown, select Create a new key pair.
  2. Enter a Key pair name.
  3. Click Download Key Pair and store it in a secure location.
    Important: You will not be able to download the key file again after it is created.
  4. Click Launch Instances

Launch CloudBolt

  1. ​Find your instance in the AWS Console EC2 Instances dashboard and click its Instance ID for details.
  2. Under Public IPv4 address, click Open Address.
  3. At this stage there will not be a valid certificate on your appliance so bypass the security warning in your browser to continue.
  4. Enter the CloudBolt License you received from your CloudBolt account team or contact sales@cloudbolt.io.
  5. Continue with Guided Setup.
