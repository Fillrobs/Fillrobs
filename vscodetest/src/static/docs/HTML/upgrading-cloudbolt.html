Upgrading CloudBolt

Learn how to upgrade CloudBolt to a new version.

  • Upgrade your Database to PostgreSQL
  • Upgrading to 9.X from 8.X
  • Upgrade Using a Yum Repository
  • Maintenance Mode
  • Custom Code Analysis Upgrade Tool (CCAT)
CloudBolt upgraders are distributed as .tgz archives. They can be downloaded either from CloudBolt’s UI or from the pertinent post in the support portal at https://support.cloudbolt.io/hc/en-us. The upgrader is designed to persist your customizations, but be sure to review the Release Notes for any changes you may need to be aware of.

Overview

  1. Read the release notes.
  2. Best practice (but not required): set up the Continuous Infrastructure Testing (CIT) feature to have automated tests.
  3. Back up your CloudBolt instance by duplicating the VM or taking a snapshot.
  4. Decide to upgrade from the UI or from the command line.
If you have a multi-server CloudBolt (ex. the DB is on a different VM, it is an HA setup), then you will need to run the upgrader from the command line.

Upgrading from the Web Interface

If there is a problem with the upgrade, ssh to the CloudBolt server and become root to inspect logs, and correct the problem. If you cannot do this, find someone who can before beginning the upgrade.

  1. In CloudBolt, click Admin > Version & Upgrade Info.
  2. Click the download and upgrade button.
  3. Right click on the job link to open it in a new tab.
  4. Watch the job to ensure the download succeeds and the upgrade is initiated. The progress of the upgrade will not be visible on this page after CloudBolt enters maintenance mode.
  5. Switch back to the Version & Upgrade Info page to track the upgrade progress after CB enters maintenance mode.

Upgrading from the Command Line

When upgrading an external database, ensure mySQL service is disabled before running the upgrader on the app server. Use systemctl disable mysql. When upgrading an external database, ensure the database service is disabled before running the upgrader on the app server. For MySQL, use systemctl disable mysql and for PostgreSQL, use systemctl disable postgresql-14.
  1. Download the upgrader tgz by either:
    1. In CloudBolt, select Admin > Version & Upgrade Info. Then select Download X.X.X upgrader to Server
    2. Download the upgrader tgz from the CloudBolt support portal and copy to the CloudBolt server under /var/tmp
  2. In a command line interface, ssh as root (ssh root@ip.address.here) to the CloudBolt server and run cd /var/tmp
  3. Untar the file by running the command tar xzf name_of_cloudbolt_file.tgz (for example: tar xzf cloudbolt_upgrader_3.0-src-f893cd2.tgz)
  4. Go into the directory by running cd cloudbolt_upgrader*
  5. If the database that this CloudBolt instance is connecting to is on a different server, edit the install_config file to specify the mysql_host or postgresql_host
  6. Run the command ./upgrade_cloudbolt.sh
  7. CloudBolt will upgrade to the new version.

Verifying the Upgrade

  1. Click on the CloudBolt logo at the bottom of any page, and verify the version is as expected.
  2. Run common operations that users depend on from your CloudBolt to ensure they function properly.
  3. Alternately, you can use CloudBolt’s Continuous Infrastructure Testing feature to test common orders and actions in an automated fashion.
