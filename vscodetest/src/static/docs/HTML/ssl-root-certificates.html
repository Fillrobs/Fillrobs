SSL Root Certificates in CloudBolt

Learn how to add Certificates to CloudBolt's Trusted Certificate Authorities


In order for CloudBolt to verify TLS/SSL connections which have a certificate issued by a private Certificate Authority it is necessary to add the private Certificate Authority to CloudBolt's built-in list of trusted SSL Root Certificates.

CloudBolt ships including the certifi (Mozilla curated) bundle of trusted Certificate authorities so connections to public cloud services should not require SSL Root certificates to be added.

If you are using an SSL Intercepting proxy server to access public cloud services, you will need to update your SSL Root Certificates with your proxy CA.
Follow the steps to find the root certificate of your proxy server CA using a browser that is configured to use the same proxy as the CloudBolt appliance.


Enabling SSL Verification

SSL verification can be toggled globally on this page but it can also be toggled on the connection information for some resource handlers.

Ensure you add all necessary private CA certificates to the SSL Root Certificates before enabling SSL Verification.

If you are developing a plugin and want to enable SSL verification, see Development Tip for making requests using the CloudBolt CA bundle.

Adding CA Certificates in CloudBolt

CloudBolt recommends adding the CA Root certificates to the SSL Root Certificates instead of the leaf certificates. This serves two purposes:

  1. When certificates are renewed CloudBolt functionality will not be impacted
  2. Limits the number of certificates that need to be maintained if internal services are issued from the same private/internal CA.

Add the Root Certificate(s) in CloudBolt

  1. In CloudBolt, on the SSL Root Certificates dashboard, click + New Root Certificate
  2. Open your the certificate (Base-64 encoded X.509 or PEM) to be added with a text editor or follow the Find the Root Certificate steps.
  3. Copy the entire contents of the certificate file (including the -----BEGIN CERTIFICATE------ and END lines)​
  4. Paste in the format certificate in the box.
  5. Click Create

The Root Certificate is now trusted by CloudBolt. Repeat for additional Root Certificates.

Find the Root Certificate(s)

If you do not have the CA certificate you can follow these instructions if you have the HTTPS url that is failing to verify.

The instructions to find the Root Certificates depends on your operating system. Choose Mac or PC below according to your workstation type.

[+] PC
[-] PC

PC

  1. In a Browser navigate to the HTTPS url that you need CloudBolt to verify
    e.g. https://vcenter.example.com
  2. Click the padlock icon to the left of the URL.
  3. Click Certificate to open the Certificate details.
  4. Click the Certification Path tab.
  5. Double click the certificate at the top of the certificate hierarchy (the Root):
     
  6. On the Root certificate, click Details tab.
  7. Click Copy to File...
  8. Select the Base-64 encoded X.509 (.CER) format and Export to a folder or Desktop.
  9. Navigate to the exported Certificate. Right click and open it in a text editor.
[+] Mac
[-] Mac

Mac

  1. In a Browser navigate to the HTTPS url that you need CloudBolt to verify
    e.g. https://vcenter.example.com
  2. Click the padlock icon to the left of the URL.
  3. Click Certificate.
  4. Navigate to the Root (top) level certificate.

  1. Click the large Certificate icon and drag it to the folder of your choice or Desktop for easy access.
  2. Find the file and right click > Open With > Keychain Access.
  3. Add the Certificate to the Keychain.
  4. Find the certificate in the list (filtering by category Certificates to narrow down the list)
  5. Right click the Certificate and click Export "The Certificate Name"...
  6. In the File Format dropdown, select Privacy Enhanced Mail (.pem).
  7. Navigate to the exported Certificate. Right click the .pem file and open it in a text editor.
