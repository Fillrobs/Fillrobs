CloudBolt Concepts

Familiarize yourself with CloudBolt's key concepts.

CloudBolt models your enterprise cloud using the following core building blocks:

Concept Description
Resource Handlers A Resource Handler is a connection to your existing Clouds or virtualization tools. VMware, AWS, Azure, and many more are supported.
Environments An Environment represents a grouping of resources. An Environment could represent a cluster or region of a Resource Handler (Cloud), a subset of VM images and networks with associated configuration parameters. 
Servers Servers represent Virtual Machines (VMs) or Managed Physical Servers. Every server belongs to one Environment and one Group.
Groups Groups are a way to organize CloudBolt users. Groups facilitate access to Environments, Blueprints, resources, reports, servers, and many other objects.
Catalog The Catalog is where users order Blueprints (Services) defined by the CloudBolt administrators.
Blueprints Blueprints simplify complex orchestration into orderable components (e.g. creating an S3 bucket). Learn more about Blueprints.
Orchestration Orchestration enables CloudBolt administrators to trigger CloudBolt plugins/scripts to perform actions automatically on defined events.
Networks Networks define the subnets and vlans that can be made available to Servers in CloudBolt from those available to your Resource Handlers.

External Systems

CloudBolt integrates with a number of types of infrastructure technologies which can form part of provisioning orchestration actions or self-service operations by CloudBolt end-users.

Connection Type Description
Configuration Managers Enable CloudBolt to install software and manage the configuration of servers with tools like Puppet, Ansible or Chef.
Container Orchestrators Enable CloudBolt to manage containerized objects like Pods and Services on Kubernetes.
Provisioning Engines Enable CloudBolt to use Razor to install a OS on a bare VM or a physical server.
IT Service Management Enable CloudBolt to use integrate with ServiceNow's Catalog or approvals
Load Balancers Enable CloudBolt to configure F5 BIG-IP LTM as part of an application deployment.
IPAM Enable CloudBolt to assign IPs using BlueCat, Infoblox, phpIPAM or Solarwinds
Data Protection Enable CloudBolt to protect workloads using Rubrik or Cohesity


In addition, CloudBolt introduces several novel tools and concepts that make managing your enterprise cloud even easier:

  • A high-performance Job Engine that works under the hood to process your orders, orchestrations, and ad-hoc management.
  • Continuous Infrastructure Testing performs end-to-end tests of your Blueprints on a routine basis to ensure your available services are ready to be provisioned, providing valuable insight into the health of your (provisioning) infrastructure.
  • Rules perform if-this-then-that checks on a routine basis, performing such actions as quota enforcement, lab clean-up, and anything you can model with a Python script.
  • A RESTful CloudBolt API v2.0 and internal Python API as well as samples in the product and on the CloudBolt Forge.
  • Rates provide a technology-agnostic way of modelling charge-back or show-back for your infrastructure based on criteria and rates that you define, without being tied to a specific technology’s pricing structure.
  • Inventory and cost Reports built into the product that make it easier to invoice and communicate the scale of your infrastructure.
