Upgrade Notes

When upgrading CloudBolt, please review all Upgrade Notes between the version of CloudBolt that you are currently running and the target version that you are upgrading to.

Important:

Customers upgrading from a version older than 9.4.6 will have to perform one of the following steps to install the mysqlclient python package.

  1. Manually install the mysqlclient when prompted, and click Y after successfully installing to continue with the upgrade.
  2. Upgrade to release 9.4.6 prior to upgrading to 9.4.7. This is required for the following reasons:
    • As of release 9.4.7, CloudBolt no longer ships with the mysqlclient for license compliancy reasons.
    • In release 9.4.6, CloudBolt was upgraded from Python 3.6 to 3.9.4. As part of the upgrade all 3.6 packages are removed and the equivalent 3.9.4 are installed. Therefore, a customer who upgrades from an earlier release than 9.4.6, the python 3.6 mysqlclient is removed and the 3.9.4 equivalent is not installed.
      • All custom Python packages will need to be re-installed.
    • The mysqlclient is required for customers to successfully upgrade to release 9.4.7.
Upgrade Notes
  • Custom Code: As of v9.4.6, the return values of the find_key_material and get_file_for_key_material internal methods have changed from: (i) 3-tuple to a 2-tuple and (ii) 2-tuple to simply a string, respectively. If either methods are in Custom Code (Actions and UI Extensions), please update accordingly.
  • API Browser: The API Browser and root view will now default to API v3 instead of v2. To view the complete list of API v3 endpoints, append /api/v3/cmp to your instance IP.
  • Power Schedules: Changes to the Power Schedule functionality resulted in updates to the Auto-power control Servers and Auto-power control Resources CloudBolt Actions Plug-ins. If you have modified these Plug-ins, please plan on integrating these changes with yours. We recommend using the CCAT to identify areas that have been modified.

Deprecation Warnings

The following features have been deprecated from CloudBolt v9.4.7 
  • Providing SSH Key material by placing a file in the proserv directory was no longer supported as of v9.4.6.
    • Please upload the material via the SSH Keys tab on the Resource Handler in the UI.
  • Netscaler advanced network support functionality.
  • Google Compute Engine functionality has been replaced by GCP.

Upcoming Deprecation Warnings

The following features will be deprecating in the next upcoming release:
  • The key_material Parameter (CustomField)
    • This will mean that it can no longer be used to store private key material for SSH Keys, having been replaced by SSHKey objects.
    • If you are using the key_material Parameter directly in custom code, you will need to update it. The CustomField and all of its values will also be deleted at that point, which will remove any old private keys stored in the values; you may wish to check them for any values you want to save.
    • SSH Keys that have been managed through the UI should roll over to the new approach automatically with no manual steps needed.
  • The global_key_name Parameter (CustomField), since it will be replaced by the general key_name Parameter and the use of SSHKey objects.
    • If you are using the global_key_name Parameter directly in custom code, you will need to update it in order to continue to use those SSH Keys. The CustomField and all of its values will also be deleted at that point.
    • SSH Keys that have been managed through the UI should roll over to the new approach automatically with no manual steps needed.

New Version Highlights

CommVault Data Protection Service Provider

  • CommVault Backup and Recovery
  • The CloudBolt CommVault DPF Service Provider integration provides quick and simple access to data protection capabilities offered by CommVault Backup & Recovery. These capabilities include the core Data Protection capabilities of Protection Plans, Snapshots, and Recovery.

Metal-As-A-Service (MAAS) Virtual Machine Resource Handler

  • MAAS Virtual Machine Selection
  • This feature work extends the MAAS Resource Handler to provision MAAS-managed virtual machines based on KVM (not LXD), in addition to the bare metal capabilities already in the Resource Handler.
  • Multiple disk sizes can be requested at provisioning time.

Simultaneous Appliance (OneFuse and CMP)

  • Run OneFuse and CloudBolt CMP on the same appliance, switching between the two with the application of a product license. This enhances the experience of customers who have licenses for both products; users no longer have to reconfigure or configure two appliances in order to use both products.
    • License OneFuse in CMP
    • Grant Users Access to OneFuse
  • CloudBolt OneFuse will adopt CMP's versioning scheme with the release of 9.4.7.
  • As part of the Simultaneous Appliance work, the EULA has been updated to be available to the user to accept regardless of which product is licensed first. 

PostgreSQL Database Upgrade

Starting with 9.4.7, the product will no longer ship with MySQL operating system packages and MySQL Python packages.
  • Updating your Database to PostgreSQL
  • Starting with release CloudBolt 9.4.7 CloudBolt will ship with PostgreSQL version 13 as the backend database, replacing MySQL as the preferred database backend.
  • Both MySQL and PostgreSQL will be supported for customers who upgrade to 9.4.7.

Content Library Updates

  • Content Library has gotten a UI and UX refresh, including:
  • Improved card layout.
  • Cards now show last updated date.
  • Removed hover on cards.
  • Added a download button on the card.
  • Moved Categories filters to the side.
  • Added a sorting order.
  • Added a slide-out panel for when content is selected.
  • Added the option to add images to expanded content.

Additional Improvements

Enhancements

General Enhancements

  • The Disk tab will still show on Resource Handlers that don't support Disk modifications. DEV-19911
  • Removed vCenter supported version text from Guided Setup. The documentation will act as source of truth for supported vCenter versions. DEV-20247
  • Added the group.manage_members_view_unassigned_users Permission that allows a User to view and add Users to Groups they manage that do not currently have a Group assigned to them (such as new Users). Add this permission to the Group Admin Role if desired. DEV-19380
  • Added Digest Algorithm and Signing Algorithm options to the SSO Setup form. Options are SHA1 and SHA256. DEV-20487

Job Engine

  • Allow the Job Engine to forcibly restart itself when Job(s) exist that have been running 2x longer than the max timeout. See Enable the Job Engine to Forcibly Restart. DEV-19656
    • If the restart is successful, the Job Engine will run again as normal.
    • If the restart is not successful and Jobs continue to run up to and past 3x the max global timeout, an email will be sent to the CB Admins with a description of the Job that caused the issue.

Orchestration

  • Updated the out-of-the-box Blueprint Validation Orchestration Action script so that Blueprints with errors show context of the error on the Order Form. If you have previously changed the Blueprint Validation script from the default, you will be able to compare your current script to the latest default, and revert it to the default if desired. DEV-19989

Power Schedules for Servers and Resources

  • Set the timezone for a Power Schedule from the Edit Power Control Schedule window. DEV-18884
    • These changes involve updates to two Recurring Job Actions: 1) Auto-power control servers and 2) Auto-power control resources.
      If any manual code changes were made on either of these Actions, then you will need to manually integrate our changes into the Actions' custom code. Please reach out to Customer Success for assistance.
  • The default time zone set in the CloudBolt installer has been changed to UTC. DEV-20356
  • The functionality that attempted to adjust displayed times based on timezone request data from the browser has been removed. DEV-20356

Provisioning
