Infrastructure Health Checking with CloudBolt CIT

Learn how to use Best Practices to verify a Resource Handler's connection with CloudBolt's Continuous Infrastructure Testing (CIT) feature.

Introduction

As a CloudBolt Admin, it is important to check and verify the health of overall infrastructure that your CloudBolt Server is connected with, before an end user finds the issue. 

Similarly, when a CloudBolt Admin creates a Blueprint for the end users to use, testing that Blueprint is crucial.

CIT is a capability of CloudBolt that helps Admins test public and private cloud connections, remote scripts, provisioning jobs, and more. For setup and use, see CloudBolt's Continuous Infrastructure Testing documentation.

Prerequisites

  • You are a Super Admin user in CloudBolt.
  • A Resource Handler is already configured in CloudBolt. See Resource Handlers.
  • You have the Resource Handler ID.

Verify Resource Handler Connection via CIT

Resource Handlers are considered the starting point when provisioning a Server and running a Remote Script using Tech Specific Script Execution from CloudBolt. Programmatically verifying and checking the health of a Resource Handler's connection is one Best Practice that a CloudBolt Admin should consider.


Create a New CIT Test

  1. Log in to CloudBolt as a Super Admin user.
  2. Click Admin > Orchestration > Continuous Infrastructure Testing.
  3. Select + New Action Test.
  4. Click CloudBolt Plug-in > + Add a New CloudBolt Plug-in.
  5. Enter a Name, and scroll to the bottom of the form.
  6. Make sure Enter code later radio button is selected for File Location.
  7. Click Create.


Update the Action Code



  1. Locate the test you just created under the Tests list.
  2. Click the Test Name link.
  3. In Actions, click the Name of your test to open the Plug-in for editing.
  4. Replace the default code with the following:

Copy Code
Python
from common.methods import set_progress
from resourcehandlers.models import ResourceHandler
from utilities.logger import ThreadLogger

logger = ThreadLogger(__name__)



def run(*args, **kwargs):
    rh = ResourceHandler.objects.get(id=int("{{ resource_handler }}"))
    rh = rh.cast()
    if rh.can_verify_connection:
        set_progress(f"Verifying connection to '{rh}'.")
        rh.verify_connection()
        set_progress(f"Successfully verified connection to '{rh}'.")


Add the Resource Handler ID

  1. Navigate back to the Continuous Infrastructure Testing dashboard.
  2. Under Tests, click the [] Pencil Icon in the Action column.
  3. Enter the Resource Handler ID in the Default Value for Resource Handler field.
  4. Click Save Changes.

Verify the New Test

  1. On the Continuous Infrastructure Testing dashboard, click Execute Tests.
  2. Click the Test Name to see its details, including the status.

Outcomes

A Job will be created, and you will see either a successful or a failed run, depending on the health of the Resource Handler's Connection.
