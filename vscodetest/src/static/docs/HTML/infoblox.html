Infoblox

Infoblox is an IP Address Management platform. Use CloudBolt's Infoblox IPAM integration to manage IP assignment, DHCP reservation, DNS resolution, and other IP management related tasks using this integration.


Infoblox Setup Overview

  1. In CloudBolt, click Admin > Integrations > IP Management.
  2. Click Add an IPAM…
  3. Enter the appropriate credentials for your Infoblox instance.
  4. Click Create.
  5. Import IPAM Network CIDRs and assign them to Resource Handler Networks.
  6. Enable the Infoblox orchestration for this instance from the Orchestration tab.


Infoblox Overview Tab

From the Overview tab, view and edit current credentials, verify the connection, or remove the connection.


Maximum # of Network Results

Edit the Maximum # of Network Results field to increase the number of allowed importable Networks from Infoblox.

If your Infoblox instance has a very high number of Networks, you can set a maximum number of Network results from the Overview tab. If there are more Networks in your Infoblox instance than allowed here, CloudBolt will display an error when attempting to import them.

Set the Maximum # of Network Results to a number higher than the total amount of Infoblox Networks in your instance.
  1. On the Overview tab, click the Pencil icon next to Maximum # of Network Results.
  2. Enter a number between 1 and 2,147,483,647. The default value is 5,000.
  3. Click Save.


Infoblox Networks Tab

Associate Infoblox Networks with Existing CloudBolt Networks. From the Networks tab of your Infoblox instance, you can map a single Infoblox network CIDR to one or more Resource Handler networks. 

Any CloudBolt networks that are so mapped will delegate hostname validation and IP address allocation to the given Infoblox server.

  1. Navigate to the Networks tab and click Import IPAM Networks.
  2. Select the Networks you would like to import and click Import selected networks.
  3. In the Resource Handler Networks column, click the Pencil icon.
  4. Choose one or more Resource Handler Networks to associate to this IPAM Network.
  5. Click Save.

Infoblox Orchestration Tab

On the Orchestration Tab, find and edit a sample Orchestration hook for Infoblox that defines functions to:

  • Validate a hostname 
  • Allocate an IP for a hostname 
  • Setup DHCP for a host 
  • Restart the Infoblox DHCP service 
  • Delete a host

You may wish to edit these functions for more sophisticated logic. Enable or disable Orchestration using the toggle.

Troubleshooting Tips

CloudBolt needs to have a domain name configured when deploying a new Server to and Infoblox-associated network. This can come from Parameters on the order or from the, DNS Domain value of the CloudBolt Network, itself. That domain also needs to be what’s configured in Infoblox, or it will refuse to register a host, the first step in the Infoblox workflow for allocating an IP. For similar reasons, the CloudBolt network also needs a Gateway and Netmask defined.

[LEGACY] Enable the Infoblox related plug-in actions

Please note that if you are using the, “IPAM Associations,” approach, none of the steps or plugins marked in this document as, “[LEGACY]” are necessary.

CloudBolt ships with a few sample plug-ins that can interact with Infoblox server registered with Cloudbolt on the previous section. The use case included in the samples consists of pre-allocating IP addresses, and creating DNS and DHCP entries for a host in Infoblox. The VM being created is setup for DHCP but the IP it gets can be considered static for all intents and purposes, because the MAC is registered with Infoblox’s DHCP to always return the preallocated IP. If you are using the combination of Infoblox + VMware and would like to configure the VMs for static IP, please contact us.

The sample plug-ins make use of specific CloudBolt parameters that should be manually created if you want to use this sample integration.

[LEGACY] Required Parameters for the Sample Plug-ins

Create these two parameters in CloudBolt (in the Admin –> Parameters part of the UI) before enabling the Infoblox actions:

  • Host FQDN (label: ‘Host FQDN’, name: ‘host_fqdn’, type: ‘String’) - the fully qualified domain name that will be registered in the Infoblox DNS.
  • Infoblox IP (label: ‘Infoblox IP’, name: ‘infoblox_ip’, type:’String’) - Stores the IP assigned to the host record in Infoblox. This is set on the server record in CloudBolt prior to application installation.

[LEGACY] Infoblox 01 - Calculate FQDN and Allocate IP (Trigger point: Pre-Create Resource)

Enable this plug-in if you want to allocate an IP with Infoblox for the server being provisioned. This plug-in will use the value of the host_fqdn parameter on the server if it is set. If not, the plug-in will combine the hostname of the server and the dns_domain and set this on the server as host_fqdn.

[LEGACY] Infoblox 02 - Setup DHCP for Host (Trigger point: Pre-Network Configuration)

Enable this plug-in action if you have you server configured for DHCP and want to make sure the Infoblox controlled DHCP server returns the IP allocated in the previous action.

[LEGACY] Infoblox 03 - Make sure server IP matches Infoblox (Trigger point: Pre-Application)

Enable this plug-in if you have a configuration manager (such as Chef) that needs line of sight to your server and you need to set the IP on the server record in CloudBolt to the IP that was allocated by Infoblox.

[LEGACY] Infoblox 04 - Delete Host Record (Trigger point: Post-Decommission)

Enable this plug-in if you want CloudBolt to make API calls to Infoblox to delete a host record and free up the IP allocation whenever a server is deleted using Cloudbolt.
