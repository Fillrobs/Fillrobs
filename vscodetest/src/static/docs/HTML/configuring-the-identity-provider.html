Configuring the CloudBolt Service Provider on the IdP

Configure the IdP with the parameters expected and provided by the SSO Provider you configured on the CloudBolt instance. Most likely, you will need to send your CloudBolt Service Provider’s metadata to your IdP.


Select your Identity Provider from the list below for setup steps for that IdP.

  • Azure AD Identity Provider
  • Office365 Identity Provider
  • Okta Identity Provider
  • OneLogin Identity Provider
  • PingOne for Enterprise Identity Provider


Required Configured Fields

The following fields will need to be included on your IdP configuration. The IdP should extract some or all of these from CloudBolt SP metadata:

  • Public Key (certificate) if you want communications between CloudBolt and your IdP to be signed.
  • Name ID Format to tell the IdP and CloudBolt what format user names/IDs must be in.
  • Required Attributes and Attribute Name fields must be configured so that the IdP will send the right attributes with the expected field names to CloudBolt.

    For example, if CloudBolt has these settings on the SSO Provider:

    • Required Attributes: Email, Username
    • Email Attribute Name: userEmail
    • User ID Attribute Name: userName
    Then the IdP must be configured to send a User’s email in a field named “userEmail” and the User’s username in a field named “userName”.

Metadata

The CloudBolt Service Provider will need the “Identity Provider metadata” information.

On the Overview tab for a given CloudBolt SSO Provider, there are two links:

  • The Service Provider’s Metadata URL / Entity ID.
  • Assertion Consumer Service (ACS) URL (displayed as Single Sign-On URL).
Success Message Your IdP may allow you to simply input the metadata URL, or you may need to visit the metadata URL and copy it to a field on your IdP.

Upload and Download Metadata

Upload IdP Metadata XML

Upload the metadata xml collected from your SSO Identity Provider instance.

  1. On the Single Sign-On Providers Dashboard, click the Name of the Service Provider you are configuring in CloudBolt (e.g. Azure AD).
  2. There are three ways to upload the IdP Metadata:
    1. Copy the IdP metadata text from the link into the Metadata Text field. 
    2. Copy the URL of the IdP metadata and paste into the Metadata URL field.
    3. Download the XML file and upload using the Metadata File option.

Download Metadata

Download a metadata file to provide to your SSO IdP if required. The Metadata XML file contains the Certificate and some attributes.

  1. On the Overview tab, click Download Metadata to save an XML file with the Metadata for this provider. 

Certificates

For many Service Providers, the Certificate is included in the Metadata file.

If your IdP does not allow uploading Metadata, which contains the Certificate, click Certificates to upload.
