Azure (Resource Manager)

Azure provides a new way to deploy and manage virtual machines and other resources. If you used the earlier service deployment model and want to learn about the changes, see Understanding Resource Manager deployment and classic deployment.

Set up Azure Portal in Azure

  1. Navigate to your Azure portal.
  2. From the dashboard, click Subscriptions.
  3. Copy the Subscription ID you would like CloudBolt to use. This will be used when setting up the Resource Handler in CloudBolt.
  4. Create an Active Directory application that can access resources. Follow the instructions available in Azure’s documentation.
  5. Record the Application ID, Secret, and Tenant ID for the new Active Directory application. These will be needed to set up the Resource Handler in CloudBolt. See Microsoft's documentation for help gathering these IDs.

    6. The new Azure portal refers to Client ID as ‘Application ID’, and Tenant ID as ‘Directory ID’. The Secret will be an authentication key generated using Azure’s documentation.
  6. Add the Application as an owner for the Subscription:
    1. Click on Subscriptions from the menu on the left and select the subscription you will be using. You might have to click ‘More services’ to see the option for ‘subscriptions’.
    2. Click Access control (IAM) for the chosen subscription.
    3. Click Add > Owner.
    4. Search for your application name, select it and click Save.

Required Azure Role at Server Order Time

When ordering an Azure server in CloudBolt, the user may select Auto-create new Resource Group or Use an Existing Resource Group.

These options require different permissions:

Resource Group Type Required Azure Role
Auto-Create New Resource Group Contributor or higher
Use an Existing Resource Group

The following three roles, if Owner is not allowed for your subscription:

  • Network Contributor
  • Virtual Machine Contributor
  • Storage Account Contributor

Set up the Azure Resource Handler

The Azure Resource Handler allows users to connect CloudBolt to their Azure endpoint.

To set up the Azure Resource Handler:

  1. Go to Admin > Resource Handlers.
  2. Click +Add a new resource handler...
  3. Select Azure. Fill in the required fields:
Field name Notes
Cloud Environment Select a cloud environment from the dropdown. The default is Public.
Subscription ID The Subscription ID found in the Subscription page of the Azure Portal.
Application ID Found on your application in Azure. See Azure's docs for information locating this.
Authentication Key Also called a Secret key. See Azure's docs for information on Secret or Authentication Keys. 
Directory ID Found on your application in Azure. See Azure's docs for information locating this. 
  1. Click Save
