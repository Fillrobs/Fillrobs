Release Notes: 9.4.1

As always, when upgrading CloudBolt, please review all Notes between the version of CloudBolt that you are currently using and the version to which you're upgrading.       

Major Improvements

Network Virtualization

  • CloudBolt supports network virtualization with the integration of NSX-T Data Center. NSX-T is a virtualized networking platform that programmatically creates and manages software-based virtual networks. CloudBolt's NSX-T integration allows users to automate the deployment of an entire network using a CloudBolt Blueprint. 

Guided Setup

  • Fresh installations of CloudBolt now have the option to use the Guided Setup feature to quickly authorize a vCenter Resource Handler, import networks/clusters, customize Environments, and jump to specific pages within CloudBolt.

Azure Shared Image Gallery

  • Improvements to the Azure Shared Image Gallery feature allow shared images to be imported for every region they've been replicated to. DEV-15698

Other Improvements

Enhancements

  • CloudBolt now supports Kerberos as a WinRM Transport type. See WinRM with Kerberos for setup. DEV-15820
  • Sub-Blueprint build items can now be configured so that even if they fail, deployment of the rest of the Blueprint will continue. DEV-14296
  • When multiple OS Builds are available on the Server Tier, Blueprint Admins can determine the initial value of OS Build presented on the Order Form, using the Initial OS Build field. DEV-14716
  • Made updates to the Recent Activity and Orders Dashboard widgets to improve user experience. DEV-16172, DEV-16165

Bug Fixes

  • Reports that should be restricted to the logged-in user were still showing in the Reports dropdown and list page. DEV-14790
  • ​An incorrect number of Roles was showing in the success message after importing Roles for Chef. DEV-15930
  • PHPAdminIPAM now respects global SSL verification. DEV-15676
  • SSH Keys from one Resource Handler could appear when setting credentials for the templates on another Resource Handler, if the two templates shared an OS Build. DEV-15514
  • The Server Stats page of an AWS Server was incorrectly displaying time zones for some stats. DEV-14758
  • Dependent fields were not working on bulk Server Actions. DEV-16150​
  • Dependent fields were not consistently receiving servers and server arguments. DEV-16150​

Security Improvements

There are no security fixes associated with v9.4.1.
