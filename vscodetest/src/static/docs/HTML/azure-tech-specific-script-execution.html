Azure Tech-Specific Script Execution

Learn how to use Tech-Specific Script Execution to run Remote Scripts on Azure.

Introduction

Azure Tech-Specific Script Execution allows users to run Remote Scripts on Azure when CloudBolt cannot directly access or run commands on the virtual machine being targeted. This accomplished using the Run Command feature of Azure.

Restrictions by OS Type

  • Run PowerShell Scripts in a Windows VM in Azure
  • Run Shell Scripts in a Linux VM in Azure

Setting up Azure Tech-Specific Script Execution

Prerequisites

  • An Azure Resource Handler with an Environment.
  • An Azure account with virtual machines on which you will be running remote scripts.

Setup Overview

  1. Add the Tech-Specific Script Execution Parameter to:
    • A Group / Environment and/or Blueprint.
  2. Provision a Server. 

Add the Tech-Specific Parameter to Group / Environment

The Parameter can also be added to individual Servers. The following steps are still required in order to enable Remote Script.
  1. Navigate to the Group or Environment that this Azure Resource Handler is associated with.
  2. On the Parameters tab, click + Add
  3. ​Find Use Tech-Specific Script Execution and add it to the Group or Environment.
  4. On the Parameters tab, click the Use Tech-Specific Script Execution parameter you just added.
  5. Ensure that Show on Objects and Available to all servers are enabled.



(Optionally) Set up a Blueprint to use Tech-Specific Execution

You can add the Parameter at the Blueprint level as well; however, as long as the Parameter is available to the Group or Environment, it'll show up on the Order Form.

  1. In CloudBolt, click Catalog and open a Blueprint on which you will enable Tech Specific Execution.
  2. Click the Parameters tab. 
  3. Add the following Parameters: Use Tech-Specific Script Execution​.
Provision a Server with Tech-Specific Script Execution
  1. Use a Blueprint to provision a server.
    • As long as the Tech-Specific Script Execution Parameter is available to the Group or Environment, it will appear on the Order Form.
  2. On the provisioned server, verify that Tech-Specific Script Execution works by running an Ad-Hoc Script.

Outcome

The Ad-Hoc script should successfully run without entering a username/password. I f Tech-Specific Script Execution is correctly enabled, the username and password fields won't be displayed on the Ad-Hoc Script form at all.

stdout and stderr are available in the output for Linux shell scripts.
