Release Notes: 9.4.2

As always, when upgrading CloudBolt, please review all Notes between the version of CloudBolt that you are currently using and the version to which you're upgrading.       

         
Upgrade Notes
  • The out-of-the-box “Resize VM” Action for Azure has had changes made to its code to simplify and better handle Action Inputs. If you have changed it, you will need to update your version to reflect the new out-of-the-box code. DEV-13708
  • Upgrade to 9.4.2 to restore the Ad-Hoc script functionality for remote scripts. DEV-16788

Major Improvements

Guided Setup 

Improvements to Guided Setup:

  • Select and set up AWS and Azure providers.
  • Use the Back button to return to the previous setup step in vCenter screen if desired.
  • Quick Setup is now a part of Guided Setup to first time users of CloudBolt.

Key Support for SSH

Activate RDP/SSH Key authentication for a given server. When enabled, a private key is used in place of a username and password.

Other Improvements

Enhancements

  • Use the Back button to return to the previous setup step in vCenter Guided Setup. DEV-16709
  • The Content Library now includes a Tintri Blueprint. Download this Blueprint to create Tintri snapshots from the UI natively and navigate between different snapshots. DEV-13071
  • Adding/Editing Networks on a Group or Environment now supports up to 10 NICs out-of-the-box. DEV-15380
  • Added the ability to disable cancelling orphan jobs in Jobengine. DEV-16893

Bug Fixes

  • Improved error handling for the AWS Rate Hook when an OS Build does not have an OS Family. DEV-16106
  • References to a Resource were not getting populated in hostname templates. DEV-16689
  • When exporting a Blueprint with one or more server build items, any associated Environment Selection Orchestration is not included in the export. DEV-15355
  • Improved rate card estimates for Azure node sizes. DEV-16784
  • Azure SSL certification verification was failing during connection validation of the Resource Handler. DEV-16712
  • Attempting to reset a user's password did not save the new password and would not allow access to CloudBolt. DEV-16824
  • Remote Scripts would not run when the OS Family of a Server did not exactly match the OS Families configured on the Remote Script (e.g. when the Remote Script was configured for Linux and the Server was a Linux sub-type like CentOS). DEV-16788
  • OpenStack snapshots were silently failing even though the Job in CloudBolt showed a successful snapshot creation. DEV-16768
  • Clicking Save at the end of the password reset process did not route the user to the CloudBolt Dashboard. DEV-16824
  • Some Password Reset Question answers were returning as incorrect when they were correct. DEV-16931
  • The Password Reset Link in a user's email did not work if the CloudBolt Portal did not have a set domain. DEV-16942

Security Improvements

  • There are no security fixes associated with v9.4.2.
