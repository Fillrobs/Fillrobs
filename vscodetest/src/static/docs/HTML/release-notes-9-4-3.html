Release Notes: 9.4.3 - Roth

As always, when upgrading CloudBolt, please review all Notes between the version of CloudBolt that you are currently using and the version to which you're upgrading.       

Major Improvements

Dashboard Improvements

  • The Dashboard widgets respect user permissions. This means that widgets will only show information for, and provide interactivity for users who have the correct permissions for those actions.
  • The Reports widget received a makeover and now has the same look and feel as the other Dashboard widgets.
  • Dashboard widgets can be added, removed, rearranged, and hidden on a per-user basis.
  • Rates on the Servers widget on the Dashboard now reflect the currency set in Admin > Rates. DEV-16846

Other Improvements

Enhancements

  • Increased the text character limit on all Resource Handler IP and Name fields. DEV-16950
  • The source of GCP Rates has been updated. Note: Google Cloud Platform controls rate accessibility and CloudBolt may not have access to the most recent rates. DEV-16804
  • Improved parameter validation when ordering a Blueprint through the API. CloudBolt checks if a field is both "required" and provided by the user before checking if it satisfies constraints regarding related fields. DEV-16946

Bug Fixes

  • supervisord was incorrectly attributing the jobengine.log to root:root instead of apache:apache. This resulted in the jobengine.log no longer being available to the httpd process. DEV-15662
  • SSO: Duplicate error on login when an IdP sent different/updated user first name, last name, or email address. DEV-15819
  • Deleting a server with an assigned IP Address (in IP Pools) was not releasing the Resource and was making the IP Address unavailable for future use. DEV-15385
  • Orders that require two approvers were showing as "Pending" after the first approval, but the jobs in the order were executing before a second approval. DEV-16924
  • Generated Parameter Options action "Net Options as Env/Group Intersection" was still allowing access to networks associated to an Environment, even when the users were in different Groups from the assigned network(s). DEV-15628
  • VMware provisioning jobs for CentOS were always using the root u  ser regardless of the username parameter, when provisioning a CentOS template. DEV-14136
  • SSO configurations set to different portals in CloudBolt were all showing the currently browsed URL instead of the URL configured in Portals. DEV-17052
  • Some Terraform plans with multiple lines of code and input type of "string" were failing with an error. DEV-16890
  • OpenStack Resource Handler Import projects was failing when there were duplicate project names across domains. DEV-16985
  • The API sporadically failed to honor all filters when returning certain objects. DEV-16203
  • A parent Blueprint ordered through the API returned an error when the parent had a sub-Blueprint marked as "not orderable". DEV-16140
  • Order failure emails displayed the wrong order URL when sent from a portal that was not the default. DEV-16470
  • When a new CloudBolt portal was added with no LDAP Domains selected, the Login page would display a dropdown showing Local options and any LDAP servers. Now, the Login page will show no dropdown options when there is only one credential store selected (either LDAP or Local). DEV-16696
  • The CloudBolt Upgrader now rebuilds the Python binary on every Upgrade. For most customers this should not impact the upgrade process. DEV-17181

Security Improvements

  • Improved security when making an API call to filter a Server with a password attribute. DEV-16969
