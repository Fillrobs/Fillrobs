Create a Terraform Plan Action

Learn how to create a Terraform Action and add Plan files.


Terraform Actions

Before adding a Terraform Plan in CloudBolt, ensure there is at least one existing Terraform Plan Action.


Check for an Existing Terraform Plan Action

  1. In CloudBolt, click Admin > Actions.
  2. Search for Terraform.
  3. The list of Plug-in Actions will show, followed by the Terraform Plan Action if it exists.
  4. Skip to Upload a Terraform Plan if the Plan Action exists. Follow Create a Terraform Plan Action below if one does not exist.


Create a Terraform Plan Action

  1. In CloudBolt, click Catalog > Blueprints > Create New Blueprint, or open the Blueprint from which you will run Terraform Actions.
  2. In the Blueprint, click the Build tab.
  3. Click + Action
  4. In the Add Action Blueprint Item window, click Terraform Plan.
  5. Click + Create a new Terraform Plan action now
  6. Enter a Name for this Plan, then follow Upload a Terraform Plan.
When creating a new Terraform plan, specify the full path to your plan directory under plan path, i.e. /var/opt/cloudbolt/terraform/plans/

Upload a Terraform Configuration

There are four ways that you can add a Terraform configuration:

  • Fetch a git repository from a URL.
  • Upload a config as a zip file.
  • Fetch a zip file containing configurations from a URL.
  • Directory on the CloudBolt server's filesystem.


Fetching a Git Repository

You can use a HTTP(s) URL with an optional sub-directory as the source of a Terraform Plan. When the Terraform Plan Action is created (or edited) the git repository is shallow-cloned onto the CloudBolt host.

For authentication, you can either encode the credentials in the URL, or create a CloudBolt Connection Info for the given host using the Git Connection Info form.

  1. The first option looks like this: https://my-user:my-git-auth-token@github.com/myco/my-terraform-plans.git. Note this only works with the .git file type, not .tf.
  2. The Connection Info option requires the following fields to be filled out:
  • Name (Any name, the IP and Protocol are used to automatically pick this up)
  • Host (e.g., github.com)
  • Protocol (e.g., https)
  • Username (e.g., my-user)
  • Auth Token (e.g., my-git-password-or-token)
  • Label terraform-plan-source. This label is required. The ConnectionInfo may include other labels, but it must include this label.
If multiple Connection Infos with the same host have the terraform-plan-source label, the protocol field is used to filter further. If multiple Connection Infos are still found, CloudBolt chooses the first one based on Connection Info ID. 

This feature only supports HTTPS and HTTP for repositories requiring authentication. (e.g., https://github.com/myco/my-terraform-plans.git)

This feature does not support SSH git sources. (e.g., git@github.com:myco/terraform-plans.git)

Uploading zip files

You can add zipped Terraform Plans in two ways:

  • Upload a zip file.
  • Fetch a zip from a URL.

When uploading a zip file, you have the option to point to a subdirectory within the compressed directory, specified under Plan Files Subdirectory in the form. This is optional, and CloudBolt will use the directory created by extracting the zip by default as the plan directory.

From the UI, these options will show up under ‘Plan Files’ on the form for adding a new plan. By default, CloudBolt assumes that you will upload a zip file containing plan files within it.

Example

Copy Code
Code
my_plan.zip 


|- some-stuff.tf 


|- my-other-stuff.tf

    Add plan files to the local filesystem
  

  

    CloudBolt recommends adding Terraform plans via one of the above ZIP file workflows. Otherwise, you may take advantage of this option to add plans directly to the filesystem if you have a very complicated plan or are unable to compress your plan files.
  

  

    Create a directory in /var/opt/cloudbolt/terraform/plans/ and add plan files there. If your Terraform files are hosted on source control, you may clone your repo here.
  

  

    
Copy Code
Code
cd /var/opt/cloudbolt/terraform/plans/ 
mkdir <new-plan-directory> 
cd <new-plan-directory> 
scp <your-remote-plan-files> . 
# or 
git clone <plan-repo>
