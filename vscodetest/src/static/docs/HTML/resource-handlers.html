Resource Handlers

  • Creating New Resource Handlers
  • Resource Handler ID
  • Deleting Resource Handlers
A Resource Handler is a connection to your virtualization platform or public cloud provider, like AWS, Azure, GCP, and more.

Use Resource Handlers to perform actions on your Public Cloud Provider from CloudBolt, like Server creation, deprovisioning, or modification. 

Create Resource Handlers for each of your virtualization technologies and public cloud providers to use them through CloudBolt's unified interface.

Creating New Resource Handlers

Creating new Resource Handlers in CloudBolt does not change external systems in any way. It can be considered a safe, read-only operation. Changes will only be made if users are given permissions in CloudBolt to manage servers and then they make changes, for example pressing the “Power Off” button on a server.

Only CloudBolt Admins can create new Resource Handlers.

  1. Click Admin > Resource Handlers > Add a resource handler... 


  1. Select your technology from the list.
  2. Enter the required information for your chosen technology. See the individual Resource Handler pages for details.
  3. Click Continue.
  4. Enter any required information, then click Create.

Resource Handler ID

The Resource Handler ID is a unique number assigned to a Resource Handler when it is created. This value is used when running CIT tests to verify the health of a Resource Handler's connection.

  1. In CloudBolt, click Admin > Resource Handlers.
  2. Click the Name of the Resource Handler to open it.
  3. In the URL bar, note the numeral appended to the end of the URL.

It will look like: https://yourcloudboltIP/admin/resourcehandlers/3/. In this example, 3 is the Resource Handler ID.

Deleting Resource Handlers

Deleting a resource handler in CloudBolt will mark the associated virtual machines as historical, but will not affect your virtual machines or other resources in the hypervisor or public cloud environment.

Environments associated with a deleted resource handler, however, will be deleted from CloudBolt, as will other objects created in CloudBolt to represent resources in the public cloud (such as imported networks). Nothing in the actual cloud environment will be affected.
