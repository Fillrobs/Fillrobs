Development Tips for Plug-ins

Learn about advanced options developers can use to customize CloudBolt Plug-ins.

Running Remote Scripts using Connection Info Objects

Connection Info objects can be used to run remote scripts on legacy technology that may not have a REST API. Another use case is to run commands remotely on Windows Servers using the WinRM transport or Linux Servers using SSH.

Windows Servers

  1. Create a Connection Info object in CloudBolt's UI and specify the protocol as winrm. 
  2. In the CloudBolt CLI (shell_plus), execute the following commands to specify the transport type. Supported types include ntlm, kerberos, credssp, or basic depending on your requirements.
Copy Code
Python
from utilities.models import ConnectionInfo
connection_info = ConnectionInfo.objects.get(name='my_connection_info')
connection_info.transport = 'ntlm'
connection_info.save()


Linux Servers

  1. Create a Connection Info object in CloudBolt's UI  and specify the protocol as ssh.

Sample

Below is an example of executing a script on the host of your Connection Info object. This sample works for both Windows and Linux servers.

Copy Code
Python
script_contents= 'echo Hello World'
connection_info.execute_script(
    script_path=None,
    script_contents=None,
    script_args=None,
    timeout=120,
    file_extension=None,
    runas_username=None,
    runas_password=None,
    runas_key=None,
    run_with_sudo=False,
)


Using Stored Credentials

If you need to use credentials in a plug-in, one way to store them securely is to use Connection Info. Go to Admin > Connection Info and create a Connection Info entry with an easily identifiable name and the information you need. Then in your plug-in you can access and make use of the Connection Info using something like the following code snippet.

Copy Code
Python
connection_info = ConnectionInfo.objects.get(name='my_connection_info')
ip = connection_info.ip
username = connection_info.username
password = connection_info.password

Making requests using the Requests library

Requests is a common library for making HTTP requests in Python. When making requests to HTTPS endpoints, the SSL certificates will be verified against a list of Root Certificates provided by the Certifi library. CloudBolt also provides a way to append additional certificates to the default list, or to disable verification altogether if the CloudBolt installation requires it. Those settings are available at Admin SSL Root Certificates. CloudBolt provides a method for fetching the appropriate settings, which can be used in plug-ins that would work across CloudBolt instances.

Copy Code
Python
import requests
from utilities.helpers import get_ssl_verification
response = requests.get('https://github.com', verify=get_ssl_verification())
