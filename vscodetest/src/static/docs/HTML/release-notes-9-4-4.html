Release Notes: 9.4.4

As always, when upgrading CloudBolt, please review all Notes between the version of CloudBolt that you are currently using and the version to which you're upgrading.

Upgrade Notes
  • The TECH_SPECIFIC_SCRIPT_EXECUTION global setting is now set to False out of the box. To re-enable SSM globally, setTECH_SPECIFIC_SCRIPT_EXECUTION = True` in /var/opt/cloudbolt/proserv/customer_settings.py. To enable tech-specific script execution on a per-use-case, add the `use_tech_specific_execution` parameter to Blueprints and Servers.
  • If you find that your LDAP Domain(s) have disappeared as options on the login screen, go to the relevant Portal(s) at Admin > Branding and Portals to edit and add the desired Domain(s) to make them be listed as options again.

Major Features

Server Summary

Each page that has Server data now displays pie charts and bar graphs that represent Server statistics. See the Server Summary section of the documentation linked above for details.

  • Power Status (On/Off)
  • Installed OS
  • Jobs in a Complete state (Success, Warning, and Failure)
  • Server Status
  • Resource Handlers
  • Owners
  • Groups
  • Environments

Terraform Plan Orchestration Actions

Customize the way Terraform is called and how resources are integrated with CloudBolt. 

  • Terraform integration in CloudBolt is now exposed in Orchestration Actions, allowing Administrators to customize how Terraform is invoked and how CloudBolt integrates and manages Terraform-provisioned resources.
  • Added the ability to refresh Terraform Plans from a remote source.
  • Improved administration of Terraform Actions, making the mapping between Terraform Variables and CloudBolt Action Inputs more robust and easier to manage.
  • Updated the packaged version of Terraform from 0.12.18 to 0.13.5

AWS Tech-Specific Script Execution

AWS Tech-Specific Script Execution allows users to run Remote Scripts on AWS when CloudBolt cannot directly access or run commands on the virtual machine being targeted. This accomplished by setting AWS System Manager (SSM) and running Remote Scripts using the Run Command feature of AWS.


CloudBolt on Oracle Enterprise Linux Support

  • For those who prefer Oracle Enterprise Linux (OEL) to CentOS, we have created a Knowledge Base article for converting an existing OEL 7 server into a CloudBolt Appliance. 

Other Improvements

Enhancements

  • Tech-specific script execution is now available for AWS via AWS Systems Manager. DEV-14138
  • Server Events created as part of a Job will now be associated to that Job's Owner. This matches the existing behavior of Resource Events. Previously, Server Events created as part of a Job had no associated Owner. DEV-15994
  • Added Resource Handler to the options for Customize Server List Columns. DEV-16048
  • Optimized the performance of "Group Server Cost (Details)" and "Group Server Cost (Summary)" reports. DEV-15620
  • Updated the Server Cost Detail UI Extension in the Content Library to fix an issue with the Costs tab on a Server. DEV-13943 
  • Updated the list of available Node Sizes when ordering an Azure server with “Premium_LRS” storage. DEV-17012
  • Added the ability to disable the password challenge question for all users using the Password Reset Question Miscellaneous Setting option. DEV-17099 
  • Added the ability to duplicate an existing User Role, copying its associated permissions. DEV-13713
  • vSphere Content Library template deploy updates: DEV-14648
    • Deploy without Resource Pool selection (directly to Cluster).
    • Added CloudBolt generated annotation/notes.
    • Deploys to folder defined by Resource Handler/Parameter.

Bug Fixes

  • Re-importing Terraform Plans would remove customizations to Terraform Plan Action Inputs. DEV-17067
  • Server History and Resource event messages showed an anomalous 'b' character. DEV-15994
  • Server Action names longer than 40 characters displayed incorrectly. DEV-16811
  • Some existing Azure private images could not be used for provisioning. DEV-17258
  • Fixed a resource leak that caused some resource-intensive jobs to be terminated before completion. DEV-17213
  • The New Server button was still visible even when the New Server button Miscellaneous Setting was disabled. DEV-16869
  • Information about a Resource was being omitted from a Server Tier's Hostname Preview field on the Order Form. DEV-15405

Security Improvements

  • Fixed a discrepancy in how Role Based Access Control (RBAC) was applied between UI and API users. DEV-17403
