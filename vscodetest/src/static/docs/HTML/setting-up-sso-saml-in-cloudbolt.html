Setting up SSO/SAML in CloudBolt

First, set up the SSO IdP in CloudBolt.

  1. In CloudBolt, click Admin > Security > Single Sign-On (SSO).
  2. Click + Add a Single Sign-On IdP
  3. ​Select the IdP type.
The type only affects how the IdP shows up in the CloudBolt UI and the initial settings for some of the options. All of the IdP types can be configured to work as any of the others.
  1. Complete the Configure a SAML Provider Form.
  2. Click Create.
  3. Configure CloudBolt in the IdP.
  4. Input the IdP metadata via the Configure Using field and Metadata URL, File, or Text fields. 

Configure a SAML Provider Form

Field name Notes
Name  (Required) A name for this SSO provider that will be displayed to users.
Name ID Format (Required) The format of the Name/ID that CloudBolt will send to and expect from this IdP. Use the format that is accepted by your IdP.
Metadata Validity Limit (Required) The number of hours that this configuration is expected to be accurate. IdP consuming the metadata of this SSO Provider may cache metadata for the number of hours specified in this field before refreshing.
Accepted Time Difference (Required) Maximum time difference allowed between the SSO Provider and the IdP (in seconds).
Contact Person Designate a contact person to administrate this SSO configuration. Limited to CB Admin role Users. The user must already be configured in CloudBolt to select them from the dropdown.
Organization Name Simple name of this organization. This name is used in CloudBolt's metadata and when generating a certificate.
Organization Display Name Full name of this organization. This name is used in CloudBolt's metadata and when generating a certificate.
Organization URL Full URL for this organization. This URL is used in CloudBolt's metadata and when generating a certificate.
Branded Portal Select a Branded Portal to associate this SSO Provider with. A Branded Portal must be selected to ensure SSO works correctly.
Sign Requests If checked, CloudBolt will cryptographically sign requests sent to this IdP.
Signing Algorithm
Algorithm used for the Signing Value in the SAML request to the IDP. Options are SHA1 and SHA256.
Digest AlgorithmAlgorithm used for the Digest Value in the SAML request to the IDP. Options are SHA1 and SHA256.
Force Authentication If checked, the user will be required to log in to the IdP again, even if they were already logged in.
Assertion Signed If checked, this SSO provider will require sent assertions to be signed.
Response Signed If checked, this SSO provider will required Authentication Responses to be signed.
Allow Unknown Attributes If checked, this SSO provider will accept unknown attributes from the IdP. Check to allow additional attributes other than username, email, first_name, and last_name from the IdP.
Create Unknown Users CloudBolt recommends keeping this box checked. This indicates if CloudBolt should create a new User if one cannot be found that matches a valid assertion from the IdP. Unchecking this box means manually creating users.
Debug If checked, the SSO provider will log extra information and deliver a verbose error log.
Required Attributes The fields on the User model that CloudBolt uses to get or create the user coming from the IdP, as well as determine User uniqueness.
Email, Given Name, Surname, User ID Attribute Name These fields map User attributes on CloudBolt to the expected field name on an incoming IdP request. For example, if Email Attribute Name is "userEmail" on this form, then the IdP attribute to send users' emails must also be named "userEmail".
