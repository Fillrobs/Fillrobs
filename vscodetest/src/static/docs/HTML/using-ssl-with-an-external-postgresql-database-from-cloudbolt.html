Use SSL with an External PostgreSQL Database from CloudBolt


  • PostgreSQL Database Configuration
  • CloudBolt Database Configuration

PostgreSQL Database Configuration

Perform the following on the PostgreSQL Server.

  1. Add the following to the PostgreSQL data directory: /var/lib/pgsql/14/data
    • Server SSL Certificate: server.crt
    • Private Key: server.key
  2. Update the SSL configuration to enable SSL:
    1. Edit /var/lib/pgsql/14/data/postgresql.conf:
      1. Uncomment ssl and set the value to on
    2. If the filename for Server SSL Certificate is not server.crt and/or Private Key is not server.key, perform the following:
      1. Uncomment ssl_cert_file and set the value to the Server SSL Certificate filename if it is not server.crt
      2. Uncomment ssl_key_file and set the value to the Private Key filename if it is not server.key

    Example /var/lib/pgsql/14/data/postgresql.conf
    Copy Code
    Code
    # - SSL -

    ssl = on
    #ssl_ca_file = ''
    #ssl_cert_file = 'server.crt'
    #ssl_crl_file = ''
    #ssl_crl_dir = ''
    #ssl_key_file = 'server.key'
    #ssl_ciphers = 'HIGH:MEDIUM:+3DES:!aNULL' # allowed SSL ciphers
    #ssl_prefer_server_ciphers = on
    #ssl_ecdh_curve = 'prime256v1'
    #ssl_min_protocol_version = 'TLSv1.2'
    #ssl_max_protocol_version = ''
    #ssl_dh_params_file = ''
    #ssl_passphrase_command = ''
    #ssl_passphrase_command_supports_reload = off

  3. Update the allowed hosts to use SSL by adding hostssl entries for your IP/IP Ranges:

    1. Edit /var/lib/pgsql/14/data/pg_hba.conf:

      Example /var/lib/pgsql/14/data/pg_hba.conf
      Copy Code
      Code
      # TYPE  DATABASE        USER            ADDRESS                 METHOD

      # "local" is for Unix domain socket connections only
      local   all             all                                     peer
      # IPv4 local connections:
      host    all             all             127.0.0.1/32            scram-sha-256
      # IPv6 local connections:
      host    all             all             ::1/128                 scram-sha-256
      # Allow replication connections from localhost, by a user with the
      # replication privilege.
      local   replication     all                                     peer
      host    replication     all             127.0.0.1/32            scram-sha-256
      host    replication     all             ::1/128                 scram-sha-256
      hostssl all             all             192.168.56.0/24         scram-sha-256

  4. Restart the database
    Copy Code
    Code
    systemctl restart postgresql-14

  5. Verify SSL is configured properly by connecting to the database using psql command below:
    Copy Code
    Code
    #Replace value for password in the psql command below

    psql "dbname=cloudbolt host=db.cloudbolt.vm user=cb_dba password=yourdbpassword sslmode=require"

    Example expected return after executing psql command above
    Copy Code
    Code
    # Example of successful psql command return
    psql (14.2)
    SSL connection (protocol: TLSv1.2, cipher: ECDHE-RSA-AES256-GCM-SHA384, bits: 256, compression: off)
    Type "help" for help.

    cloudbolt=>

For further detailed information, refer to PostgreSQL documentation: Secure TCP/IP Connections with SSL.

CloudBolt Database Configuration

  1. Add the following files to the CloudBolt Database directory: /var/opt/cloudbolt/proserv/db
    • server.crt
    • server.key
    • root.crt
  2. Change ownership of all 3 files to apache:apache with the following commands:
    Copy Code
    Code
    chown apache:apache /var/opt/cloudbolt/proserv/db/server.crt
    chown apache:apache /var/opt/cloudbolt/proserv/db/server.key
    chown apache:apache /var/opt/cloudbolt/proserv/db/root.crt

  3. For all Web nodes and all Job Engine nodes, update the your database settings:

    1. Edit /var/opt/cloudbolt/proserv/custom_settings.py to add the OPTIONS for SSL:

      Example /var/opt/cloudbolt/proserv/custom_settings.py with OPTIONS
      Copy Code
      Code
      #Replace value for PASSWORD in the block below

      DATABASES = {
          'default': {
              "ENGINE": "django.db.backends.postgresql_psycopg2",
              "NAME": "cloudbolt",
              "USER": "cb_dba",
              "PASSWORD": "yourpassword",
              "HOST": "db.cloudbolt.vm",
              "PORT": "5432",
              'OPTIONS': {
                  'sslmode': "require",
                  'sslcert': "/var/opt/cloudbolt/proserv/db/server.crt",
                  'sslkey': "/var/opt/cloudbolt/proserv/db/server.key",
                  'sslrootcert': "/var/opt/cloudbolt/proserv/db/root.crt",
              },
          }
      }

  4. In a web browser, log in to the CloudBolt UI and navigate to a couple of different pages.
  5. On the database server, connect to the database using psql and execute the following query.
    Copy Code
    Code
    #Replace value for password in the psql command below

    psql "dbname=cloudbolt host=db.cloudbolt.vm user=cb_dba password=yourdbpassword sslmode=require"
    psql (14.2)
    SSL connection (protocol: TLSv1.2, cipher: ECDHE-RSA-AES256-GCM-SHA384, bits: 256, compression: off)
    Type "help" for help.

    cloudbolt=> SELECT * FROM pg_stat_ssl;
