CloudBolt Content Library

Use the CloudBolt Content Library to import custom orchestration content into different areas of your CloudBolt instance.


Get Started

  1. In CloudBolt, click the Library caret from the main menu or click the Library heading to open the Library page.
  2. Click Library Page to see the Content Library dashboard, or select a link on the right to filter results.
  3. Select a Content Library item to import and click its card.
  4. Click Download . Review the information in the text box, then click Import.
  5. The downloaded content will open in Overview mode, ready to configure.

Content Library Categories

Blueprints

Download and deploy Blueprints from the Content Library to run custom workflows on existing CloudBolt-provisioned VMs. Downloaded Blueprints are imported into your Catalog for use.

  1. In the Content Library, select the Blueprints tab, and choose a Blueprint.
  2. Click Download. Review the information in the text box, then click Import.
  3. The Blueprint will open in a new browser tab, ready to configure.
Blueprints are imported along with all of their Actions and Sub-Blueprints. The Blueprint will not be deployable initially and may require additional configuration. Server Tiers may need to be assigned an appropriate OS Build. Environments may need to be configured, and Parameter Options predefined if desired.

UI Extensions

Download UI Extensions in CloudBolt to extend CloudBolt's out-of-the-box capabilities. Examples include Veeam backups, Scheduled Snapshots, and Tintri storage array support.

  1. In the Content Library, select the UI Extensions tab, and choose a XUI.
  2. Click Download. Review the information in the text box, then click Import.
  3. The UI Extension will open in a new browser tab, ready to configure.
The UI Extension Package will be imported along with all of its views and templates. It may be necessary to restart the webserver for this package to be fully operational.
If an existing extension package has the same name, it will be replaced with the imported one. This cannot be undone.

Server Actions

Download custom Server Actions to enable on a CloudBolt Server details area.

  1. In the Content Library, select the Server Actions tab, and choose a Server Action.
  2. Click Download. Review the information in the text box, then click Import.
  3. The Server Action will open in a new browser tab, ready to configure.

After import, configure the Server Action to suit your requirements. For example, constrain actions by Group, Environment, or technology; customize default values; and customize action code.

Resource Actions

Download custom Resource Actions to add arbitrary actions to deployed resources, via buttons on their details view.

  1. In the Content Library, select the Resource Actions tab, and choose a Resource Action.
  2. Click Download. Review the information in the text box, then click Import.
  3. The Resource Action will open in a new browser tab, ready to configure.
After import, configure the Resource Action to suit your requirements. For example, constrain actions by Group, Environment, or technology; customize default values; and customize action code.

Orchestration Actions

Download custom Orchestration Actions to hook your custom logic into CloudBolt processes at various trigger points, e.g. during server ordering or provisioning.

  1. In the Content Library, select the Orchestration Actions tab, and choose an Orchestration Action.
  2. Click Download. Review the information in the text box, then click Import.
  3. The Orchestration Action will open in a new browser tab, ready to configure.
After import, configure the Orchestration Action to suit your requirements. For example, constrain actions by Group, Environment, or technology; customize default values; and customize action code.

Recurring Jobs

Download custom Recurring Jobs to run any Job or Action on a user-defined recurring schedule.

  1. In the Content Library, select the Recurring Jobs tab, and choose a Recurring Job.
  2. Click Download. Review the information in the text box, then click Import.
  3. The Recurring Job will open in a new browser tab, ready to configure.

Rules

Rules allow CloudBolt admins to define conditional logic that is periodically executed and that may trigger custom actions based on the result.
