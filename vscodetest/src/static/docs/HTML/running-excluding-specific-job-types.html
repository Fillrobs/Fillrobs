Running/Excluding Specific Job Types

CloudBolt provides environment variables that can be used to restrict a Job Engine to run only certain types of jobs by calling them by their attribute name

  • CBJE_INCLUDE_JOB_TYPES=<type 1>, <type 2>, ... , <type n>: Runs only jobs of the given types
  • CBJE_EXCLUDE_JOB_TYPES=<type 1>, <type 2>, ... , <type n>: Excludes all jobs of the given types from running

Ex: CBJE_INCLUDE_JOB_TYPES=provision,expire,decom

To find what type a certain job is, enter shell_plus by entering this command in your cloudbolt directory at opt/cloudbolt/ 

./manage.py shell_plus

and then get the type of a given job using the following commands 

j = Job.objects.get(pk=<JOB_ID>) j.type

Where JOB_ID is the number at the end of the header on the desired job’s detail page

The available job types are as follows:

Type Name Associated Scenarios
provision Provisioning a server
decom Deleting/decomissioning a server
healthcheck Performing a health check on the CloudBolt instance
expire Setting and checking servers for expiration
servermodification Modifying a server in any way
deploy_blueprint Deploying Blueprints
install_pod Installing pods
installapplications Installing applications
uninstallapplications Uninstalling applications
syncvms Synchronizing VMs with their Resource Handlers
sync_svrs_from_pe Synchronizing Servers with their Provision Engines
functionaltest Jobs spawned from Continuous Infrastructure Testing
runautomations Executing Rules
runautomationactions Executing Actions within a Rule
sync_users_from_ldap Synchronizing Users from an LDAP server
run_flow Running an Orchestration Flow
install_apps_with_connector Installing applications with a Configuration Manager such as Chef, Anisble and Puppet
manage_nics Managing NICs
orchestration_hook Running an Orchestration Action
delete_snapshots Deleting a Snapshot
