Outscale Resource Handler

Learn about the Outscale Resource Handler and how to set it up in CloudBolt.


3DS OUTSCALE offers Infrastructure as a Service (IaaS) solutions: on-demand computing infrastructure. IaaS users install their own applications and data in the provided virtual environments. The Outscale Resource Handler allows users to connect CloudBolt to their Outscale endpoint.

Setup Overview

  1. Enable the Outscale Resource Handler in your CloudBolt.
  2. Set up the Outscale Resource Handler.
  3. Create a Blueprint to order Outscale Resources.
  4. Provision using Outscale.

Enabling Outscale in CloudBolt

The Outscale Resource Handler must be enabled in the CloudBolt UI before it can be set up.

  1. In a command-line interface, SSH into your CloudBolt instance using ssh root@<your.cloudbolt.ip>.
  2. Navigate to the proserv folder: cd /var/opt/cloudbolt/proserv
  3. Edit the customer_settings.py file. In this example, we are using vi.
    • Add the following to the top of the file: 
    • Copy Code
      Code
      from django.conf import settings

      settings.FEATURE_REGISTRY["resourcehandler::outscale"] = True
    • Use esc: then wq to save and quit.
  4. Run systemctl restart httpd
  5. Run cd /opt/cloudbolt/
  6. Run ./initialize/create_objects.py

Refresh your CloudBolt instance. You should now have the Outscale Resource Handler available in the Resource Handler list.

Setting up the Outscale Resource Handler

Set up one Outscale Resource Handler for each Region in your Outscale instance. The Regions have unique Access and Secret Keys.

To set up the Outscale Resource Handler:

  1. Go to Admin > Resource Handlers.
  2. Click +Add a new resource handler...
  3. Select Outscale.
    • Region: Enter the Region, e.g. us-east-2. The value entered here correlates to AWS-Compliant Regions. See the AWS Wiki for options.
    • Access Key: The Access Key associated with your Outscale instance.
    • Access Key Secret: The Access Key Secret associated with your Outscale instance.
  4. Click Done. Enter a Name and optional Description.

Creating the Outscale Resource Handler auto-creates an Outscale Environment and imports Outscale-specific Parameters. 

Import Images

Use the Images tab to import Images for the current Region. 

Import Networks

  1. Use the Networks tab to import Networks for the current Region.
  2. Click the Network Name to view its associated details. 

View Servers

  1. Use the Servers tab to sync VMs from Outscale.
  2. Click a Server Name to view its associated details.

Add Compute Limits

Use the Compute Limits tab to view current limits on Resources and make changes to existing limits.

Add and Link Server Tags

Tags can be linked to CloudBolt Server attributes here. Once linked, the tag's value will be synchronized with the value of the server attribute.

CloudBolt creates and tags Resources as part of the provisioning process.

  1. In the Tags tab, click + Add Tag.
  2. Enter an attribute and Tag Name. 

Tags marked as having bidirectional syncing will both update in Outscale and CloudBolt if changed in either location.

Configure the Environment

After setting up the Resource Handler, click the Environment name from the Overview tab to configure the Environment.

Add Networks

  1. On the Environment, click the Networks tab.
  2. Click [] Make changes.
  3. Click + Add a network > make your selection > Save changes.

Add OS Builds

  1. On the Environment, click the OS Builds tab.
  2. Click [] Edit > select OS Build(s) > Save.

Outscale-Specific Parameters

The Outscale Environment includes tech-specific Parameters. These are unique to Outscale and will appear on the Order Form for this Environment. Click the Edit icon in the Actions column to make changes to which Parameters should 

  • Keypair Name: Name of a key pair defined in this Resource Handler. Stored on an instance during launch to enable secure key-based access. Used instead of username/password.
  • Security Group: Name of a Security Group defined in Outscale.
  • Availability Zone: Select an Availability Zone to deploy the Resource into.
