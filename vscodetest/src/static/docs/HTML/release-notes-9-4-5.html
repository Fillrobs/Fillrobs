Release Notes

Learn about changes to the CloudBolt CMP product.


The most current version of CloudBolt is 9.4.5.1
When upgrading CloudBolt, please review all Upgrade Notes between the version of CloudBolt that you are currently running and the target version that you are upgrading to. 

CloudBolt 9.4.5 - Roth

Upgrade Notes

  • Customers upgrading to 9.4.5 will see a new permission (blueprint.any_group_can_deploy) added to any Role that currently has blueprint.manage associated to it (i.e. Resource Admin out-of-the-box, but any Role you added blueprint.manage to). Carefully consider keeping the blueprint.any_group_can_deploy Permission associated to Roles.
    • This permission allows Users to check the "Any group can deploy box" on the Group Permissions Edit window, which then allows any Group, present or future, to order the Blueprint.
  • Announcing the intent to deprecate a list of little-used features as of the next version, v9.4.6. See Deprecation Warning for the list.
  • AzureARMHandler.tenant_id was changed to azure_tenant_id in 9.4.5. This may affect customers who have custom scripts.
  • If you find that your LDAP Domain(s) have disappeared as options on the login screen, go to the relevant Portal(s) at Admin > Branding and Portals to edit and add the desired Domain(s) to make them be listed as options again.

Major Features

API v3

  • Use CloudBolt's new API (v3) to interact with CloudBolt and its Resources. See CloudBolt API v3.
  • Combined both v3 CMP and OneFuse user API into common api /api/v3/users.

Custom Code Analysis Upgrade Tool

  • Run a tool to analyze the code in your current CloudBolt instance to see areas that could be impacted during an upgrade. See Custom Code Analysis Tool.

Catalog UI Improvements

  • The Catalog UI received an overhaul, including new layout, search and filtering, and sorting options.

Scheduled Snapshots

  • Use a UI Extension and packaged Recurring Job to schedule snapshots in CloudBolt for VMware vCenter and Nutanix servers. Designate a schedule using cron-style scheduling, and set a max number of snapshots. Create manual snapshots at any time outside of the set schedule by using + Create Manual Snapshot. See Snapshots.

Microsoft Teams Support for Multi-channel Alerts

  • Microsoft Teams users can now set up Multi-channel Alerts to send notifications in designated channels. See Setting up Microsoft Teams to use Multi-channel Alerts.

Other Improvements

Enhancements

  • Added Billing and Rate support for AWS GovCloud Resource Handler. See AWS GovCloud. DEV-17464
  • Restrict New Environments in Miscellaneous Settings is now Enabled by default. This setting assigns new Environments only to the Unassigned Group instead of automatically assigning the Environment to all Groups when the Environment is created. DEV-17218
  • Improved behavior of regenerated options dependent fields on the order form in the case of a deep dependency chain. DEV-17328
  • Guided Setup now shows the names of Environments created at the end of the Environment creation step. DEV-17298
  • Improved iFrame support for ServiceNow, including two new Miscellaneous Settings: Embedded Mode toggle and Embed URLs / Frame Ancestors.
  • Improved proxy support for Google Cloud Resource Handler. DEV-17659
  • The Network Adapter Type of a vCenter template in CloudBolt (found by navigating to Admin > External Systems > Network > Edit) overrides the Network Adapter Type of newly provisioned VMs in vCenter when a vCenter server is provisioned in CloudBolt. This may also override the Adapter Type on the template in vCenter. DEV-14255
  • On Blueprints, Action type Build items can be enabled or disabled using a toggle. When an Action Build item is disabled, it will not run when the Blueprint is ordered. DEV-14972
  • On an Infoblox IPAM connection, Edit the Maximum # of Network Results field to increase the number of allowed importable Networks from Infoblox. DEV-16692
  • Improved Order Form performance when submitting an Order. DEV-18152
  • By default, Azure Disk Caching Type is set to "None" for disks from 4095 GB to 64TiB. DEV-16356
  • Added the ability to set the external IP to "None" in GCP for CloudBolt. DEV-17912

Bug Fixes

  • Concurrent Provisioning Jobs in a Chef Environment were being cancelled by the Job Engine. DEV-17577
  • Standardized AMI sorting order for AWS provider in Guided Setup. DEV-17299
  • Server Summary charts were not resizing correctly based on browser size. DEV-17376
  • Sub-Blueprints can execute scripts on servers provisioned in a parent Blueprints by setting the Execution Scope toggle to “Unrestricted”. DEV-17465
  • Parameters with global defaults that were updated in the UI returned no values in shell_plus (using the get_global_default command). DEV-17684
  • Sync VMs was sporadically failing for a KVM Resource Handler. DEV-16901
  • Attempting to run a Server Action on multiple Servers with "Requires Approval" enabled was failing. DEV-16347
  • ​Provisioning Engines were experiencing issues on creation. DEV-17982
  • Attempting to edit an HPSA Provisioning Engine was generating an error. DEV-17983
  • Running Threshold Check: Order Submission Orchestration Action, net_usage() was returning 0 for mem and CPU on Google Cloud Platform orders. DEV-17008
  • Choosing a node size during provisioning resulted in CPU and Mem not populating in Server Details for GCP. DEV-16955
  • Networking details were blank for Azure servers after changing the NIC of a VM in an Azure Subscription and running an RH Sync in CloudBolt. DEV-16243
  • Action Inputs on CloudBolt Plug-ins were sometimes being reordered during code saves. DEV-17206
  • Some Actions (CB Plug-ins, Remote Scripts, and Copy File Actions) were not respecting constraints when being run within a sub-Blueprint attached to a parent Blueprint. DEV-16945

Security Improvements

  • Resolved a security issue where non-admin users could retrieve sensitive information from Servers using Actions.
  • Resolved a potential security issue with Order Forms.

Known Issues

  • Disruption in a Resource Handler's server, such as a power outage, could cause a discrepancy in Cost Detail Report rates due to syncing issues.
  • Under certain conditions, Jobs deleting Azure VMs may fail. Retrying the operation succeeds.

Deprecation Warning

The following features will be deprecated from CloudBolt as of v9.4.6. Plan appropriately if your organization uses any of the features listed below.
  • Disabling Database Browser UI accessibility.
  • Terremark Resource Handler
  • Century Link Resource Handler
  • HPSA Provisioning Engine
  • Cobbler Provisioning Engine
  • Azure Classic Resource Handler (see Microsoft's update)
  • Jasper Reports
  • HP Blade
  • HP Helion
  • RHEV Resource Handler
  • Nebula
  • Oracle Resource Handler
  • NMAP network scanner
