Configure Trusted SSL Certificates

To log in to the CloudBolt appliance for the first time, you will need to accept the self-signed certificate. To allow your browser to verify the certificate in future, configure CloudBolt with a certificate trusted by your clients.

CloudBolt requires a certificate issued from either on Online Certificate Authority or private CA (ensure all your potential clients/user devices trust this CA).

If you already have a signed certificate and key in PEM/Base 64 encoded format, update certificate on the CloudBolt appliance via SSH/SCP. Otherwise, follow steps to Generate a CSR.

  1. Copy your PEM-formatted SSL cert to /etc/pki/tls/certs/localhost.crt
  2. Copy your PEM-formatted SSL key to /etc/pki/tls/private/localhost.key
  3. Set the file permissions to the following:

    4. Copy Code
    Code
    [Code]  
    chown apache:apache /etc/pki/tls/certs/localhost.crt /etc/pki/tls/private/localhost.key   
    chmod 400 /etc/pki/tls/certs/localhost.crt /etc/pki/tls/private/localhost.key

  4. Restart CloudBolt using service httpd restart. 

Generate a CSR

If you do not have a certificate you can create a CSR to be signed by your certificate authority using the following steps on the CloudBolt appliance via SSH.

  1. Create an OpenSSL configuration file (e.g. req.conf) and fill out the details for your CSR:

    Copy Code
    Code
    [req] 
    distinguished_name = req_distinguished_name 
    req_extensions = v3_req 
    prompt = no 
    [req_distinguished_name] 
    C = US 
    ST = VA 
    L = City 
    O = YourOrganization 
    OU = YourOrganizationUnit 
    CN = cloudbolt.example.com 
    [v3_req] 
    keyUsage = keyEncipherment, dataEncipherment 
    extendedKeyUsage = serverAuth 
    subjectAltName = @alt_names 
    [alt_names] 
    DNS.1 = cloudbolt.example.com

  2. Update the hostname for your CloudBolt instance in both places (for maximum client compatibility), CN =  and DNS.1 = 
  3. Add additional hostnames as required at the end of the file if you need additional Branded Portals:

    Copy Code
    Code
    [Code]  DNS.1 = cloudbolt.example.com 
    DNS.2 = portal1.example.com 
    DNS.3 = portal2.example.net

  4. Run the following command to create the CSR and new key based on the configuration file:

    Copy Code
    Code
    openssl req -new -out cloudbolt.csr -newkey rsa:2048 -nodes -sha256 -keyout new_localhost.key -config req.conf

  5. Now submit the cloudbolt.csr to your chosen certificate authority.
  6. This command also generates a file new_localhost.key which does not need to be shared with the certificate authority and will be used when you receive the certificate file from your CA and you follow the steps in Configure Trusted SSL Certificates to install the certificate.
